## 1.  Overview  

We are asked to **prove** that the Python fragment shown in the statement satisfies the
Hoare‑style specification  

\[
\{\,P\,\}\;\text{program}\;\{\,Q\,\}
\]

where  

* **\(P\)** – the admissible input (start and end timestamps that lie inside the
  month of April, the interval is non‑empty and at least one minute long).  

* **\(Q\)** – the four numbers printed by the program are the unique
  decomposition of the elapsed time \(T\) (in seconds) into whole days,
  whole hours, whole minutes and whole seconds, each component being in its
  natural range ( \(0\le X<24,\;0\le Y<60,\;0\le Z<60\) ).

The proof follows the **standard Hoare‑logic rules**:

* **Assignment rule**  

  \[
  \frac{}{ \{\,Q[e/x]\,\}\;x:=e\;\{\,Q\,\}}
  \]

* **Sequence rule**  

  \[
  \frac{\{P\}\;S_1\;\{R\}\qquad\{R\}\;S_2\;\{Q\}}
       {\{P\}\;S_1;S_2\;\{Q\}}
  \]

* **Conditional rule** (not needed here) and **loop rule** (not needed).

The program is a **straight‑line** piece of code, therefore a chain of
assignments and a final `print`.  The only non‑trivial part is to show that the
expressions used for `segundos`, `minutos`, `horas` and `dias` compute exactly
the quantities defined in the post‑condition.  To do that we introduce a few
*auxiliary* variables that capture the “borrow’’ information used by the author
(`segundoMaior`, `minutoMaior`, `horaMaior`).  These variables are Boolean
values that are immediately turned into `0` or `1` by the `int()` conversion.
The proof will treat them as **borrow flags**.

---

## 2.  Formalisation of the program  

For readability we rename the input variables exactly as they appear in the
specification:

| Python variable | Specification symbol |
|----------------|----------------------|
| `diaA`         | \(d_s\) |
| `horaA`        | \(h_s\) |
| `minutoA`      | \(m_s\) |
| `segundoA`     | \(s_s\) |
| `diaB`         | \(d_e\) |
| `horaB`        | \(h_e\) |
| `minutoB`      | \(m_e\) |
| `segundoB`     | \(s_e\) |

The program (after the two `input` statements) consists of the following
assignments, written in Hoare‑logic order:

```text
1  segundos   := (segundoB - segundoA) % 60
2  segundoMaior := segundoA > segundoB
3  minutos   := (minutoB - minutoA - int(segundoMaior)) % 60
4  minutoMaior  := minutoA > minutoB
5  horas     := (horaB - horaA - (int(segundoMaior) or int(minutoMaior))) % 24
6  horaMaior   := horaA > horaB
7  dias      := diaB - diaA - (int(segundoMaior) or int(minutoMaior) or int(horaMaior))
```

The four `print` statements are *observational* – they do not affect the state,
so they do not participate in the Hoare proof; they merely output the values
computed in the previous assignments.

---

## 3.  Useful arithmetic facts  

We shall repeatedly use the following elementary properties of the Euclidean
remainder (`%`) and integer division (`//` in Python, but we use the mathematical
notation `⌊·⌋` and `mod`).

| Property | Formal statement |
|----------|------------------|
| **(A)**  | For any integers `a,b` with `b>0` : `a = b·⌊a/b⌋ + a mod b` and `0 ≤ a mod b < b`. |
| **(B)**  | If `c = a - b` and `c < 0` then `c % b = b + (c % b)` (the “borrow’’). |
| **(C)**  | `int(bool)` returns `1` when the Boolean is true, `0` otherwise. |
| **(D)**  | In Python the expression `x or y` evaluates to `x` if `x` is non‑zero,
            otherwise to `y`.  Since the arguments are `0` or `1`, `x or y` is
            exactly `max(x,y)`.  Consequently `int(segundoMaior) or int(minutoMaior)`
            is `1` iff **any** of the two borrow flags is true, otherwise `0`. |
| **(E)**  | `a % b` is the same as `a - b·⌊a/b⌋`.  Therefore `a % b` can be seen as
            “subtract the largest multiple of `b` that does not exceed `a`”.

These facts allow us to reason about each assignment as a *borrow‑aware*
subtraction followed by a modulo operation.

---

## 4.  Proof of each assignment  

We prove, by induction on the sequence of statements, that after statement *i*
the program variables hold the values required by the post‑condition.  The
induction invariant after step *i* is simply the definition of the elapsed time
\(T\) together with the “borrow’’ flags that have already been computed.

### 4.1  Invariant before the first assignment  

Let  

\[
\begin{aligned}
T &=
(d_e-d_s)\cdot 24\cdot60\cdot60
\;+\;(h_e-h_s)\cdot60\cdot60
\;+\;(m_e-m_s)\cdot60
\;+\;(s_e-s_s) .
\end{aligned}
\]

Because of the pre‑condition \(P\) we have \(T>0\).  
No auxiliary variables have been defined yet.

---

### 4.2  Assignment 1 – `segundos`

\[
\text{segundos} := (s_e - s_s) \bmod 60 .
\]

*Using (A)* we can write  

\[
s_e - s_s = 60\cdot q_s + \text{segundos},
\qquad 0\le \text{segundos}<60,
\]

where \(q_s = \big\lfloor\frac{s_e-s_s}{60}\big\rfloor\).  
If \(s_e\ge s_s\) then \(q_s=0\) and `segundos = s_e-s_s`.  
If \(s_e < s_s\) then \(q_s=-1\) and `segundos = 60 + (s_e-s_s)`.  
In the latter case a **borrow** of one minute is required; this borrow is
captured by the Boolean `segundoMaior` (defined in the next line).

Thus after statement 1 we have exactly the value \(Z\) required in the
post‑condition:

\[
Z = (s_e-s_s) \bmod 60 .
\]

---

### 4.3  Assignment 2 – `segundoMaior`

\[
\text{segundoMaior} := (s_s > s_e) .
\]

By definition of the Boolean, `segundoMaior` is true **iff** the start second
is larger than the end second, i.e. iff the subtraction in the previous step
required a borrow.  Using (C) we later obtain `int(segundoMaior) = 1` in that
case and `0` otherwise.

Hence after statement 2 we have a correct *borrow flag* for the seconds.

---

### 4.4  Assignment 3 – `minutos`

\[
\text{minutos} := (m_e - m_s - \text{int(segundoMaior)}) \bmod 60 .
\]

Consider the expression inside the modulo:

* If `segundoMaior` is **false** (`int=0`), we simply compute
  \((m_e-m_s) \bmod 60\).  This is the usual minute difference when no borrow
  from the seconds is needed.

* If `segundoMaior` is **true** (`int=1`), we subtract one extra minute –
  exactly the amount that was borrowed when we computed the seconds.

Thus the quantity inside the modulo is the *minute difference after accounting
for the possible second‑borrow*.  Applying (A) again we obtain

\[
\text{minutos}=Y,
\qquad 0\le Y < 60,
\]

where  

\[
Y = \bigl(m_e-m_s-\mathbf{1}_{\{s_s>s_e\}}\bigr)\bmod 60 .
\]

The integer part of the division (the number of whole minutes that were
borrowed) is stored implicitly in the next Boolean `minutoMaior`.

---

### 4.5  Assignment 4 – `minutoMaior`

\[
\text{minutoMaior} := (m_s > m_e) .
\]

Exactly as for the seconds, `minutoMaior` is true **iff** after the possible
second‑borrow the minute subtraction still needs a borrow from the hour
component.  Consequently `int(minutoMaior) = 1` precisely when a minute‑borrow
occurs.

---

### 4.6  Assignment 5 – `horas`

\[
\text{horas} := (h_e - h_s - (\text{int(segundoMaior)}\; \textbf{or}\;
                         \text{int(minutoMaior)})) \bmod 24 .
\]

The expression  

\[
\text{int(segundoMaior)}\; \textbf{or}\; \text{int(minutoMaior)}
\]

evaluates, by property **(D)**, to  

\[
\max\bigl(\text{int(segundoMaior)},\;\text{int(minutoMaior)}\bigr)
\in\{0,1\}.
\]

Thus it is `1` exactly when **any** of the two previous borrow flags is true,
otherwise `0`.  This is precisely the *hour‑borrow* that must be subtracted:

* If a second‑borrow happened, we already decreased the minute count by one;
  that may have caused a minute‑borrow, which in turn forces a hour‑borrow.
* If only a minute‑borrow happened (no second‑borrow), the hour‑borrow is also
  needed.
* If neither borrow happened, the hour subtraction is performed unchanged.

Therefore the quantity inside the modulo is the hour difference **after all
necessary borrows**.  Again using (A) we get

\[
\text{horas}=X,\qquad 0\le X<24,
\]

where  

\[
X = \bigl(h_e-h_s-\mathbf{1}_{\{s_s>s_e\}\lor\{m_s>m_e\}}\bigr)\bmod 24 .
\]

---

### 4.7  Assignment 6 – `horaMaior`

\[
\text{horaMaior} := (h_s > h_e) .
\]

`horaMaior` records whether, after all previous borrows, the hour subtraction
still needs a borrow from the day component.  Its integer value (`0` or `1`)
will be used in the final step.

---

### 4.8  Assignment 7 – `dias`

\[
\text{dias} := d_e - d_s -
               (\text{int(segundoMaior)}\; \textbf{or}\;
                \text{int(minutoMaior)}\; \textbf{or}\;
                \text{int(horaMaior)}) .
\]

The expression  

\[
\text{int(segundoMaior)}\; \textbf{or}\;
\text{int(minutoMaior)}\; \textbf{or}\;
\text{int(horaMaior)}
\]

is, by repeated use of property **(D)**, equal to  

\[
\max\bigl(\text{int(segundoMaior)},
          \text{int(minutoMaior)},
          \text{int(horaMaior)}\bigr)
\in\{0,1\}.
\]

Consequently it is `1` exactly when **any** of the three borrow flags is true,
i.e. when the subtraction of seconds, minutes **or** hours required a borrow
from the day component.  Subtracting this `1` (or `0`) from the raw day
difference yields the correct number of whole days:

\[
\text{dias}=W .
\]

Because the pre‑condition guarantees that the interval never leaves the month,
the result satisfies \(0\le W\le 29\) (the month has 30 days).

---

## 5.  Composition of the results  

Collecting the four values obtained after the last assignment we have

\[
\begin{aligned}
W &= \text{dias}                                    \\[1mm]
X &= \text{horas}                                   \\[1mm]
Y &= \text{minutos}                                 \\[1mm]
Z &= \text{segundos} .
\end{aligned}
\]

From the derivations above we know that each of them satisfies the range
constraints required by \(Q\) and that they are **exactly** the Euclidean
quotients and remainders that appear in the decomposition of the total number
of seconds \(T\) (definition of \(T\) in § 3).  Formally, using property **(A)**
repeatedly we obtain

\[
\begin{aligned}
T &= 24\cdot60\cdot60\;W \;+\; 60\cdot60\;X \;+\; 60\;Y \;+\; Z ,\\
0 &\le X < 24,\qquad
0 &\le Y < 60,\qquad
0 &\le Z < 60 .
\end{aligned}
\]

Thus the post‑condition \(Q\) holds.

---

## 6.  Hoare‑logic proof tree  

We now present the *formal* Hoare triple proof, using the assignment rule
repeatedly and the sequence rule to glue the steps together.

\[
\begin{array}{l}
\displaystyle
\{P\}
\; \text{read inputs}
\; \{\,P\wedge d_s,h_s,m_s,s_s,d_e,h_e,m_e,s_e \text{ are bound}\,\}
\\[2mm]
\displaystyle
\{\,P\wedge\cdots\,\}
\;\; \text{segundos } := (s_e-s_s) \% 60
\;\; \{\, Z = (s_e-s_s) \% 60 \,\}
\\[2mm]
\displaystyle
\{\, Z = (s_e-s_s) \% 60 \,\}
\;\; \text{segundoMaior } := (s_s > s_e)
\;\; \{\, \text{segundoMaior}= \mathbf{1}_{\{s_s>s_e\}} \,\}
\\[2mm]
\displaystyle
\{\, Z = (s_e-s_s) \% 60 ,\,
   \text{segundoMaior}= \mathbf{1}_{\{s_s>s_e\}} \,\}
\;\; \text{minutos } := (m_e-m_s-\text{int(segundoMaior)}) \% 60
\\[2mm]
\displaystyle
\{\, Y = (m_e-m_s-\mathbf{1}_{\{s_s>s_e\}}) \% 60 \,\}
\;\; \text{minutoMaior } := (m_s > m_e)
\\[2mm]
\displaystyle
\{\, Y = (m_e-m_s-\mathbf{1}_{\{s_s>s_e\}}) \% 60 ,\,
   \text{minutoMaior}= \mathbf{1}_{\{m_s>m_e\}} \,\}
\;\; \text{horas } := (h_e-h_s-
          (\text{int(segundoMaior)}\; \textbf{or}\; \text{int(minutoMaior)})) \% 24
\\[2mm]
\displaystyle
\{\, X = (h_e-h_s-\max(\mathbf{1}_{\{s_s>s_e\}},\mathbf{1}_{\{m_s>m_e\}})) \% 24 \,\}
\;\; \text{horaMaior } := (h_s > h_e)
\\[2mm]
\displaystyle
\{\, X = \dots ,\,
   \text{horaMaior}= \mathbf{1}_{\{h_s>h_e\}} \,\}
\;\; \text{dias } := d_e-d_s-
    (\text{int(segundoMaior)}\; \textbf{or}\;
     \text{int(minutoMaior)}\; \textbf{or}\;
     \text{int(horaMaior)})
\\[2mm]
\displaystyle
\{\, W = d_e-d_s-
    \max(\mathbf{1}_{\{s_s>s_e\}},\mathbf{1}_{\{m_s>m_e\}},\mathbf{1}_{\{h_s>h_e\}}) \,\}
\end{array}
\]

The final state after the seventh assignment satisfies exactly the
conjunction of the four equalities that constitute the post‑condition \(Q\).
By the **sequence rule** we can concatenate all the individual Hoare triples,
obtaining the overall triple

\[
\boxed{\{P\}\; \text{program}\; \{Q\}} .
\]

Since each step is a terminating assignment and the input routine is also
terminating, the whole program terminates for every admissible input.

---

## 7.  Conclusion  

We have:

* **Identified** the logical meaning of every line of the program, especially
  the three Boolean “borrow’’ flags.
* **Shown** that each assignment implements the Euclidean subtraction with
  borrow, followed by a modulo operation, exactly as required for the
  decomposition of a total number of seconds into days, hours, minutes and
  seconds.
* **Proved**—using the Hoare‑logic assignment and sequence rules—that the
  state after the last assignment satisfies the post‑condition \(Q\), i.e.
  the four numbers printed are the unique decomposition of the elapsed time
  \(T\).

Therefore the Python program is **correct** with respect to the given
pre‑condition \(P\) and post‑condition \(Q\).  ∎
