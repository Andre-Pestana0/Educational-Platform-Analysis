## 3.  Loop invariant

The only non‚ÄĎtrivial part of the program is the `while`‚ÄĎloop (lines‚ÄĮ7‚ÄĎ13).  
We introduce the following invariant `I(i)` that describes the state after the
first `i` elves have been processed (`0 ‚Č§ i ‚Č§ N`).

```
I(i) ‚Č°
   0 ‚Č§ i ‚Č§ N  ‚ąß
   bonecosHoras      = ő£_{k=1..i, G_k = bonecos}      H_k  ‚ąß
   arquitetosHoras   = ő£_{k=1..i, G_k = arquitetos}   H_k  ‚ąß
   musicosHoras      = ő£_{k=1..i, G_k = musicos}      H_k  ‚ąß
   desenhistasHoras  = ő£_{k=1..i, G_k = desenhistas}  H_k .
```

In words: after processing the first `i` input lines, each accumulator contains
exactly the total number of hours contributed by the elves that belong to the
corresponding group among those first `i` elves.

`I` is the classic *accumulator invariant* for a loop that sums values
conditionally on a tag (`G_i`).

## 4.  Proof that the invariant holds

We prove `I` by the standard Hoare‚ÄĎlogic rule for `while` loops:

```
{ Pre }  init;  while B do body od { Post }
```

where

* `B` is the loop guard `i < N`,
* `body` is the block lines‚ÄĮ8‚ÄĎ13,
* `init` are the statements before the loop (lines‚ÄĮ1‚ÄĎ6).

### 4.1  Initialization (`i = 0`)

Before the loop we have executed lines‚ÄĮ2‚ÄĎ5, which set all four hour‚ÄĎvariables
to `0`.  Hence

```
bonecosHoras      = ő£_{k=1..0, G_k = bonecos}      H_k = 0
arquitetosHoras   = ő£_{k=1..0, G_k = arquitetos}   H_k = 0
musicosHoras      = ő£_{k=1..0, G_k = musicos}      H_k = 0
desenhistasHoras  = ő£_{k=1..0, G_k = desenhistas} H_k = 0 .
```

Together with `i := 0` (line‚ÄĮ6) we obtain `I(0)`.  Thus the **initialisation
step** of the invariant is proved.

### 4.2  Preservation (`I(i) ‚ąß B ‚áí I(i+1)`)

Assume the invariant holds for some `i` with `i < N` (the guard is true).  
The body of the loop reads the next triple `(E_i+1 , G_i+1 , H_i+1)` and then
executes exactly one of the four assignments, depending on the value of
`G_i+1`.

*If `G_i+1 = bonecos`* :

```
bonecosHoras := bonecosHoras + H_i+1
```

All other hour‚ÄĎvariables stay unchanged.  By the induction hypothesis

```
bonecosHoras (old) = ő£_{k=1..i, G_k = bonecos} H_k .
```

Therefore after the assignment

```
bonecosHoras (new) = ő£_{k=1..i, G_k = bonecos} H_k  +  H_i+1
                  = ő£_{k=1..i+1, G_k = bonecos} H_k .
```

The other three sums are unchanged, because the corresponding groups do not
receive the new `H_i+1`.  Consequently `I(i+1)` holds.

*The three remaining cases (`arquitetos`, `musicos`, `desenhistas`) are
identical, merely swapping the group name.*  

Thus **preservation** of the invariant is proved for every possible value of
`G_i+1`.

Finally the statement `i := i + 1` (line‚ÄĮ13) updates the loop counter, so the
loop invariant is true **after** the whole body finishes.

### 4.3  Termination (`¬¨B ‚áí I(N)`)

When the loop exits we have `¬¨(i < N)`, i.e. `i = N`.  
By the invariant we obtain

```
bonecosHoras      = ő£_{k=1..N, G_k = bonecos}      H_k
arquitetosHoras   = ő£_{k=1..N, G_k = arquitetos}   H_k
musicosHoras      = ő£_{k=1..N, G_k = musicos}      H_k
desenhistasHoras  = ő£_{k=1..N, G_k = desenhistas} H_k .
```

These equalities are exactly the *group‚ÄĎwise totals* that appear in the
post‚ÄĎcondition.

Using the equalities proved in ¬ß4.3, the computation of `P` (lines‚ÄĮ15‚ÄĎ18) can be
rewritten step‚ÄĎby‚ÄĎstep:

```
P

   + ‚Ćä arquitetosHoras   / req(arquitetos)   ‚Ćč
```

The right‚ÄĎhand side is precisely the expression appearing in the post‚ÄĎcondition

## 7.  Remarks on the specification given in the question

The post‚ÄĎcondition displayed in the question

```
P = ő£_{i=1}^{N} ‚Ćä H_i / req(G_i) ‚Ćč
```

is *equivalent* to the group‚ÄĎwise formulation proved above **iff** the hours
contributed by different elves of the *same* group are **not allowed to be
combined**.  The problem statement (and the reference solution) *does* allow
combination: the total number of hours of a group is summed first and only
then divided by the required hours per gift.  

If we keep the per‚ÄĎelf version, the program would be *incorrect* for inputs such
as:

```

elf1  bonecos 5
elf2  bonecos 5
```

because the program would output `1` (10/8 = 1) while the per‚ÄĎelf formula yields
`0`.  Therefore the correct post‚ÄĎcondition for this problem is the
*group‚ÄĎwise* one that we proved.
