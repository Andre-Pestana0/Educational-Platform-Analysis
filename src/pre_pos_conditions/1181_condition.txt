Below is a **Hoare‑logic style specification** for the program described in the statement.  
All symbols are written in conventional mathematical notation (LaTeX‑style) and the
specification contains only the **pre‑condition** \(P\) and the **post‑condition** \(Q\).

---

### 1.  Notation used in the specification  

| Symbol | Meaning |
|--------|---------|
| \(L\) | Integer read from the first line of the input. |
| \(T\) | Upper‑case character read from the second line of the input, \(T\in\{‘S’,‘M’\}\). |
| \(M\) | A two‑dimensional array \(M\in\mathbb{R}^{12\times 12}\) (12 rows, 12 columns) whose elements are read from the input, row‑wise, left‑to‑right. |
| \(n\) | The fixed dimension of the matrix, \(n = 12\). |
| \(\displaystyle S(L) \;:=\; \sum_{j=0}^{n-1} M[L][j]\) | Sum of the elements of row \(L\). |
| \(\displaystyle A(L) \;:=\; \frac{1}{n}\,S(L)\) | Arithmetic mean (average) of the elements of row \(L\). |
| \(\mathit{out}\) | The real value that the program prints (rounded to one decimal place). |
| \(\lfloor x \rceil_{1}\) | “\(x\) rounded to one digit after the decimal point”. (Any standard rounding rule may be assumed.) |

---

### 2.  Pre‑condition \(P\)

\[
\boxed{
\begin{aligned}
P \;\equiv\; &\; 0 \le L \le 11 \\[2pt]
            &\;\land\; T \in \{‘S’,‘M’\} \\[2pt]
            &\;\land\; M \in \mathbb{R}^{12\times 12}
\end{aligned}}
\]

The pre‑condition states that the program receives a **valid row index**, a **valid operation
character**, and a **complete 12 × 12 matrix of real numbers**.

---

### 3.  Post‑condition \(Q\)

\[
\boxed{
\begin{aligned}
Q \;\equiv\; &\;
\bigl(T = ‘S’ \;\Longrightarrow\; \mathit{out}= \lfloor S(L) \rceil_{1}\bigr) \\[4pt]
            &\;\land\;
\bigl(T = ‘M’ \;\Longrightarrow\; \mathit{out}= \lfloor A(L) \rceil_{1}\bigr)
\end{aligned}}
\]

In words:

* If the operation character is **‘S’** (sum), the program must output the
  **sum of the 12 elements of row \(L\)**, rounded to one decimal place.

* If the operation character is **‘M’** (average), the program must output the
  **average of the 12 elements of row \(L\)**, rounded to one decimal place.

---

### 4.  Hoare triple for the whole program  

\[
\boxed{\{P\}\; \text{Program}\; \{Q\}}
\]

where *Program* denotes the sequence of statements that:

1. reads \(L\), \(T\) and the 144 numbers into the matrix \(M\);
2. computes \(S(L)=\sum_{j=0}^{11}M[L][j]\);
3. if \(T='S'\) sets \(\mathit{out}=S(L)\), else sets \(\mathit{out}=S(L)/12\);
4. prints \(\mathit{out}\) with exactly one digit after the decimal point.

---

### 5.  Remarks (optional, not part of the Hoare triple)

* **Termination** is guaranteed because the program performs a bounded number of
  input reads (144 numbers) and a bounded loop over the fixed size \(n=12\).

* The specification is **total**: for every state satisfying \(P\) the program
  terminates in a state satisfying \(Q\).

* The rounding function \(\lfloor\cdot\rceil_{1}\) is introduced only to capture
  the required output format; the algorithm itself may compute the exact real
  value and defer rounding to the final printing step.

--- 

This completes the formal Hoare‑logic specification (pre‑ and post‑conditions) of the problem.
