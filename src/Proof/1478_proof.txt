## 1.  What we have to prove  

The program reads a (possibly empty) finite sequence  

```
n1 , n2 , … , nk   (  ni  > 0 )
```

ended by a **0** (the 0 is *not* part of the sequence).  

For every `ni` the program must print an `ni × ni` matrix **M** whose
entries are defined by  

\[
M[i][j]=\begin{cases}
1               & i=j\\[2mm]
i-j+1           & i>j\\[2mm]
j-i+1           & i<j
\end{cases}\qquad
0\le i,j < ni
\]

and after each matrix a blank line must be printed.

Formally we want to prove the Hoare triple  

\[
\{\,\text{pre}\,\}\;P\;\{\,\text{post}\,\}
\]

where  

* **pre**  : “the input consists of a (possibly empty) finite list of
  positive integers followed by a single 0”.  

* **post** : “`ordens` contains exactly the positive integers that were read,
  in the same order; for each `ordem` in `ordens` the program prints the
  matrix described above and a blank line”.

The proof proceeds by giving a Hoare invariant for every
loop, showing that the invariant holds before the loop, is preserved by
one iteration, and together with the loop‑exit condition yields the
desired property after the loop.  The proof is completely mechanical,
so we will only sketch the main steps; the reader can fill in the
routine algebraic manipulations.

--------------------------------------------------------------------

## 2.  The **reading** part  

```python
ordens = []
aux = 1
while (aux != 0):
    aux = int(input())
    if aux != 0:
        ordens.append(aux)
```

### 2.1 Loop invariant  

\[
I_{\text{read}} \; \equiv\;
\begin{array}{l}
\text{ordens} = [\,x_1,\dots ,x_m\,] \\
\wedge\;
\forall t\;(1\le t\le m \Rightarrow x_t>0)\\
\wedge\;
\text{aux}= \begin{cases}
0 &\text{if the last value read was the terminating 0}\\
y &\text{otherwise, where }y>0\text{ is the last value read}
\end{cases}
\end{array}
\]

Intuitively: *`ordens` already stores exactly the positive numbers that
have been read so far, and `aux` holds the most recent input (0
iff the input has already terminated).*

### 2.2  Initialization  

Before the first test of the `while` we have  

* `ordens = []` – the empty list, which trivially satisfies the first two
  conjuncts (there is no `t`).  
* `aux = 1` – a non‑zero value, therefore the third conjunct holds
  (`aux` is “the last value read” and it is indeed non‑zero).  

Hence `I_read` holds before the loop.

### 2.3  Preservation  

Assume `I_read` holds at the beginning of an iteration and `aux != 0`
(the loop condition).  

*The body* reads a new integer `v = int(input())` and stores it in `aux`.  

*Case 1 – `v = 0`*  

```
if v != 0:   # false
    ordens.append(v)   # not executed
```

`ordens` stays unchanged, still containing exactly the previously read
positive numbers.  `aux` becomes 0, satisfying the third conjunct of the
invariant.  Therefore `I_read` holds after the iteration.

*Case 2 – `v > 0`*  

The `if` guard is true, so `v` is appended:

```
ordens.append(v)
```

Now `ordens = [x1,…,xm , v]`; all entries are still positive, and
`aux = v > 0`.  Hence every conjunct of `I_read` is true again.

Thus the invariant is preserved.

### 2.4  Termination  

The loop stops exactly when `aux == 0`.  At that moment `I_read` tells us

* `ordens = [x1,…,xk]` where each `xi > 0`, and
* the last value read was `0`.

Consequently `ordens` is precisely the list of all positive integers that
preceded the terminating zero – exactly the required *pre‑condition* for
the next part of the program.

## 3.  The **matrix‑construction** part  

```python
for ordem in ordens:
    matriz = []
    for i in range(ordem):
        auxiliar = []
        for j in range(ordem):
            if i == j:          auxiliar.append(1)
            if i >  j:          auxiliar.append(i - j + 1)
            if i <  j:          auxiliar.append(j - i + 1)
        matriz.append(auxiliar)
```

We prove that after the three nested loops `matriz` is exactly the matrix
defined by the specification.

### 3.1  Outer `for` – “process each order”

The Python `for` loop iterates over the *already proved* list `ordens`.
No invariant is needed beyond the standard one for a `for` loop:

\[
I_{\text{outer}}(p) \equiv
\begin{array}{l}
\text{the first }p\text{ elements of }ordens\text{ have already been processed}\\
\wedge\;
\text{for each processed element }n\text{ a correct matrix has been printed}
\end{array}
\]

`p = 0` holds before the loop, and the loop body increments `p` by one,
preserving the invariant (the body builds **and prints** a correct matrix
for the current `ordem`).  When `p = |ordens|` the loop ends, meaning that
*all* orders have been processed – exactly what we need.

Hence we concentrate on the inner two loops that build a single matrix.

--------------------------------------------------------------------
### 3.2  Inner loop over rows (`i`)

```
matriz = []
for i in range(ordem):
    auxiliar = []
    ...   # inner loop over columns
    matriz.append(auxiliar)
```

#### Invariant `I_row(i)`

After the execution of the body for a particular value of `i`
(the loop variable has already been incremented to `i+1`) we have

\[
I_{\text{row}}(i) \equiv
\begin{array}{l}
\text{`matriz` contains the first }i\text{ rows of the desired matrix}\\
\wedge\;
\forall r < i\; \forall c < ordem\; 
    \bigl(\text{matriz}[r][c] = f(r,c)\bigr)
\end{array}
\]

where  

\[
f(r,c)=\begin{cases}
1      & r=c\\
r-c+1  & r>c\\
c-r+1  & r<c
\end{cases}
\]

*Initialization* (`i = 0`): `matriz` is empty, therefore the first
conjunct holds vacuously; the second conjunct is also vacuous because
there is no `r < 0`.  

*Preservation*: assume `I_row(i)` holds at the start of an iteration.
The body constructs `auxiliar` (the next row) and appends it.
If we can show that after the inner‑column loop `auxiliar[c] = f(i,c)` for
all `c`, then after the `append` we will have the first `i+1` rows correct,
i.e. `I_row(i+1)`.

Thus the remaining work is to prove the correctness of the *column* loop.

--------------------------------------------------------------------
### 3.3  Inner‑most loop over columns (`j`)

```python
auxiliar = []
for j in range(ordem):
    if i == j:          auxiliar.append(1)
    if i >  j:          auxiliar.append(i - j + 1)
    if i <  j:          auxiliar.append(j - i + 1)
```

#### Invariant `I_col(j)`

After processing the first `j` columns (`j` already incremented) we have

\[
I_{\text{col}}(j) \equiv
\forall c < j\; \bigl(\text{auxiliar}[c] = f(i,c)\bigr)
\]

*Initialization* (`j = 0`): `auxiliar` is empty, the universal statement
is true.

*Preservation*: assume `I_col(j)` holds and we are about to execute the
iteration with the current column index `j`. Exactly one of the three
`if` statements is true, because the three conditions `i==j`,
`i>j`, `i<j` are mutually exclusive and exhaustive.  
The body therefore executes **exactly one** `append` with the value

* `1`               if `i==j`  → `f(i,j)=1`
* `i-j+1`           if `i>j`   → `f(i,j)=i-j+1`
* `j-i+1`           if `i<j`   → `f(i,j)=j-i+1`

Hence after the `append` we have `auxiliar[j] = f(i,j)`.  Together with
the induction hypothesis we obtain `I_col(j+1)`.

*Termination*: when the loop ends we have `j = ordem`.  
`I_col(ordem)` tells us that **every** column `c < ordem` satisfies
`auxiliar[c] = f(i,c)`.  Consequently `auxiliar` is precisely the `i`‑th
row of the required matrix.

--------------------------------------------------------------------
### 3.4  Putting the three invariants together  

* From `I_col` we get that after the column loop `auxiliar[c]=f(i,c)` for
  all columns `c`.  
* Adding this row to `matriz` yields `I_row(i+1)`.  
* Repeating the row loop until `i = ordem` gives `I_row(ordem)`,
  i.e. `matriz` now contains **all** rows and therefore the whole matrix
  defined by `f`.  

Thus after the two nested `for` loops we have proved the **post‑condition**
for the construction part:

\[
\text{matriz}[i][j] = f(i,j) \quad\forall\,0\le i,j<\text{ordem}.
\]

--------------------------------------------------------------------

## 4.  The **printing** part  

```python
for i in range(ordem):
    for j in range(ordem):
        if j == 0:
            print(f"{matriz[i][j]:3d}", end = "")
        else:
            print(f" {matriz[i][j]:3d}", end = "")
    print()
print()
```

The loops simply iterate over the already‑proved matrix `matriz`.  
The inner `if` only decides whether a leading space must be printed;
it does **not** change the value that is printed.  Consequently the
output of the two loops is exactly the textual representation required
by the specification (each element printed with width 3, a space between
consecutive elements, a newline after each row, and a blank line after
the whole matrix).

Formally we can state a trivial invariant:

\[
I_{\text{print}}(i,j) \equiv
\text{All entries } \bigl\{\,\text{matriz}[r][c]\mid
0\le r<i\;\lor\;(r=i\land 0\le c<j)\,\bigr\}
\text{ have already been printed in the required format.}
\]

The invariant is easily shown to hold initially (`i=j=0`), to be
preserved by each iteration (the next element is printed with the same
format), and when the outer loop finishes (`i = ordem`) we have printed
the whole matrix.  The final `print()` adds the required blank line.

--------------------------------------------------------------------

## 5.  Whole‑program Hoare triple  

We now combine the three parts:

* **Reading part** establishes  

  \[
  \{\,\text{pre}\,\}\;(\text{reading code})\;
  \{\,\text{ordens}= [n_1,\dots ,n_k]\;\wedge\; n_t>0\;\}
  \]

* **Processing part** (outer `for`) together with the inner loops
  guarantees  

  \[
  \{\,\text{ordens}= [n_1,\dots ,n_k]\,\}\;
  (\text{matrix‑construction + printing})\;
  \{\,\text{for each } n_t\text{ the required matrix is printed}\,\}
  \]

* The outer `for` iterates over **all** elements of `ordens`, therefore
  after the loop finishes the post‑condition holds for **every** input
  order.

Consequently we have proved the Hoare triple

\[
\boxed{\;
\{\,\text{input is a finite list of positive integers terminated by 0}\,\}
\;
P
\;
\{\,\text{the program prints, for each integer, the matrix defined by }f\,
\}}
\]

which is exactly the specification given in the problem statement.

--------------------------------------------------------------------

## 6.  Summary of the proof steps  

| Loop (or block) | Invariant (informal) | Reason for preservation |
|-----------------|----------------------|--------------------------|
| `while aux != 0` (reading) | `ordens` holds all positive numbers already read; `aux` is the last read value (0 iff termination) | One iteration either appends a new positive number (keeps positivity) or reads the terminating 0 (sets `aux` to 0). |
| `for i in range(ordem)` (row construction) | `matriz` already contains the first `i` correct rows | Adding a row that has just been built by the column loop makes the first `i+1` rows correct. |
| `for j in range(ordem)` (column construction) | `auxiliar[0…j‑1]` already equals `f(i,·)` for those columns | Exactly one of the three mutually exclusive `if`s fires, appending `f(i,j)`. |
| `for i in range(ordem)` (printing rows) | All entries up to position `(i,j‑1)` have been printed in the required format | The next entry `matriz[i][j]` is printed with the same format, extending the printed prefix. |
| Outer `for ordem in ordens` | All orders processed up to the current one have been printed correctly | The body prints a correct matrix for the current `ordem`. |

All invariants are **true initially**, **preserved** by one loop
iteration, and together with the **negation of the loop condition**
imply the desired property after the loop.  By the rule of composition
the whole program satisfies its specification.

∎
