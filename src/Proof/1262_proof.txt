**Answer – a Hoare‑logic proof that the program computes exactly the number of
cycles required by the specification**

### 1.  The specification

For a test case we are given  

* a string `rastro ∈ {R,W}*` (the trace of operations) and  
* an integer `processos ∈ ℕ⁺` (the maximum number of consecutive reads that can be
  performed in a single *read‑cycle*).

From the statement of the problem we define the *semantic* function  

\[
\operatorname{cycles}(rastro,processos)=
\begin{array}{l}
\text{let }c:=0,\;q:=0\text{ (c = total cycles, q = length of the current read block)}\\[2mm]
\text{for each character }x\text{ of }rastro\text{ do}\\
\qquad\text{if }x='R' \text{ then } q:=q+1;\\
\qquad\qquad\text{if }q=processos\text{ then }q:=0,\;c:=c+1;\\[1mm]
\qquad\text{else if }x='W' \text{ and }q\neq0\text{ then }q:=0,\;c:=c+2;\\[1mm]
\qquad\text{else if }x='W' \text{ and }q=0\text{ then }c:=c+1;\\[2mm]
\text{after the loop, if }q\neq0\text{ then }c:=c+1;\\[2mm]
\text{return }c .
\end{array}
\]

The **post‑condition** that the program has to satisfy is therefore  

\[
\{\,\text{input }rastro,processos\;\}\;\;P\;\;\{\,ciclos=\operatorname{cycles}(rastro,processos)\,\}.
\]

The **pre‑condition** is simply that the two values are read successfully from the
input (the program aborts on EOF, which is treated separately).

### 2.  Variables used by the program

| Variable | Meaning (in the specification)                              |
|----------|--------------------------------------------------------------|
| `rastro` | the input string (unchanged during the computation)         |
| `processos` | the integer limit for a read‑cycle (unchanged)           |
| `contador` | the length `q` of the current block of consecutive `R`s   |
| `ciclos`   | the total number `c` of cycles counted so far            |
| `i`        | the loop variable that iterates over the characters of `rastro` |

### 3.  Loop invariant for the **for‑loop**

Let  

* `k` be the number of characters already processed (`0 ≤ k ≤ |rastro|`);
* `pref = rastro[0..k‑1]` the processed prefix (empty when `k=0`);
* `q` be the value of `contador`;
* `c` be the value of `ciclos`.

Define the predicate  

\[
\mathcal I(k,c,q) \;:=\;
\begin{array}{l}
c = \operatorname{cycles}(pref,processos) \\
\land\; q = \text{length of the maximal suffix of }pref\text{ consisting only of }'R'\\
\land\; 0 \le q < processos .
\end{array}
\]

Intuitively, after having examined the first `k` characters,
*`ciclos`* already contains the exact number of cycles that the
specification attributes to the processed prefix,
and *`contador`* contains the size of the still‑open block of reads
(which can never be `≥ processos`, because a block of that size would have
already been turned into a complete read‑cycle).

---------------------------------------------------------------------

### 4.  Proof that `𝓘` is a correct invariant  

We prove the three classic conditions of a loop invariant.

---

#### 4.1  **Initialization** (`k = 0`)

Before the `for`‑loop the program executes  

```python
contador = 0
ciclos   = 0
```

Hence `c = 0` and `q = 0`.  
The processed prefix is the empty word `ε`.  
According to the definition of `cycles`,  

\[
\operatorname{cycles}(ε,processos)=0 .
\]

Thus `𝓘(0,0,0)` holds.

---

#### 4.2  **Maintenance** (from `k` to `k+1`)

Assume `𝓘(k,c,q)` holds at the beginning of an iteration and let `x = rastro[k]`
be the next character.  We consider the three branches of the program.

* **Case `x = 'R'`**  

  The program does `contador += 1`, i.e. `q' = q+1`.  

  *If `q' = processos`* the program executes  

  ```python
  contador = 0
  ciclos   += 1
  ```  

  Hence `q'' = 0` and `c' = c+1`.  
  By the definition of `cycles`, a block of exactly `processos` reads
  contributes **one** read‑cycle, therefore  

  \[
  \operatorname{cycles}(pref\!·\!R,processos)
  = \operatorname{cycles}(pref,processos) + 1 .
  \]

  Consequently `c' = cycles(pref·R,processos)` and `q'' = 0`,
  i.e. `𝓘(k+1,c',q'')` holds.

  *If `q' < processos`* the program does **nothing else**, so `c' = c`,
  `q'' = q'`.  The new suffix of reads has length `q' (<processos)`,
  therefore again `𝓘(k+1,c',q'')` holds.

* **Case `x = 'W'` and `q ≠ 0`**  

  The program executes  

  ```python
  contador = 0
  ciclos   += 2
  ```  

  Hence `q'' = 0` and `c' = c+2`.  
  According to the specification, a write that follows a non‑empty block of
  reads closes that block (adding **one** cycle) **and** the write itself
  needs an additional write‑cycle, i.e. a total of two cycles.
  Thus  

  \[
  \operatorname{cycles}(pref·W,processos)
  = \operatorname{cycles}(pref,processos) + 2 .
  \]

  Therefore `𝓘(k+1,c',q'')` holds.

* **Case `x = 'W'` and `q = 0`**  

  The program executes only `ciclos += 1`.  Hence `c' = c+1`, `q'' = 0`.  
  The specification says that a write that is **not** preceded by any read
  creates exactly one write‑cycle, so again  

  \[
  \operatorname{cycles}(pref·W,processos)=\operatorname{cycles}(pref,processos)+1 .
  \]

  Hence `𝓘(k+1,c',q'')` holds.

In every possible branch the invariant is preserved, therefore the
maintenance condition is proved.

---

#### 4.3  **Termination** of the `for`‑loop  

When the loop finishes we have processed the whole string,
i.e. `k = |rastro|`.  By `𝓘` we know

\[
c = \operatorname{cycles}(rastro,processos)\quad\text{and}\quad
q = \text{length of the trailing block of reads of }rastro,
\;0\le q < procesos .
\]

If `q = 0` the program does nothing more; the final `if contador != 0`
is skipped and the value of `ciclos` already equals the required result.

If `q ≠ 0` the program executes

```python
ciclos += 1
```  

which adds exactly the missing **partial** read‑cycle that the specification
assigns to a final block of reads that never reached `processos`.  Consequently

\[
c' = c+1 = \operatorname{cycles}(rastro,processos) .
\]

Thus after the whole `for`‑loop and the final `if`, the post‑condition
`ciclos = cycles(rastro,processos)` holds.

---------------------------------------------------------------------

### 5  Whole program (outer `while True`)

The outer loop simply repeats the whole computation for each test case
until an `EOFError` is raised.  The *pre‑condition* for one iteration is
the successful reading of a line `rastro` and an integer `processos`; the
*post‑condition* of the iteration is exactly the one proved above.
When the input stream ends, the `except EOFError` branch terminates the
program.  Hence the program is *partially correct* for every finite
sequence of test cases, and it also *terminates* because each iteration
processes a finite string and the `while` loop ends only on EOF.

---------------------------------------------------------------------

### 6  Formal Hoare triple

Putting everything together we obtain the Hoare triple

\[
\boxed{
\{\,\text{input lines are available}\,\}\;
\texttt{while True: …}\;
\{\,\forall\;rastro,processos\;.\;ciclos = \operatorname{cycles}(rastro,processos)\,\}
}
\]

which has been proved by the invariant `𝓘` for the inner `for`‑loop and by
the trivial termination argument for the outer `while`.
