## 1.  What we have to prove  

We are asked to **prove** that the program  

```python
Distancia, Diametro1, Diametro2 = map(int, input().split(' '))
print("{:.2f}".format(Distancia/(Diametro1 + Diametro2)))
```  

satisfies a given **Hoare specification**  

```
{ P }  program  { Q }
```

where  

* **P** ‚Äď the pre‚ÄĎcondition that must hold before the program starts,  
* **Q** ‚Äď the post‚ÄĎcondition that must hold after the program terminates,  

and the proof must be performed with the inference rules of **Hoare Logic** (assignment, sequencing, I/O, etc.).

Below we first state a sensible specification for this particular problem, then we decompose the program into elementary commands, write a Hoare triple for each command, and finally compose the triples to obtain the overall correctness proof.

---

## 2.  Specification  

The problem (Beecrowd 2950 ‚Äď *Iniciante ‚Äď N√≠vel‚ÄĮ1*) asks the user to input three **integers**:

* `Distancia` ‚Äď the distance between two Palant√≠ri,  
* `Diametro1` ‚Äď the diameter of the first Palant√≠r,  
* `Diametro2` ‚Äď the diameter of the second Palant√≠r.

The required output is the *√≠ndice de coincid√™ncia m√©dia* (ICM), i.e.

\[
\text{ICM}= \frac{\text{Distancia}}{\text{Diametro1}+\text{Diametro2}}
\]

rounded (and displayed) with exactly two decimal digits.

Therefore a natural Hoare specification is

```
P  ‚Č°  (input consists of three integers d, a, b) ‚ąß (a + b ‚Č† 0)
Q  ‚Č°  (the program prints the value d/(a+b) with exactly two decimal
       digits, using the rounding mode of Python‚Äôs format specifier
       "{:.2f}")
```

The extra conjunct `a + b ‚Č† 0` is needed because the program performs a division; it guarantees that the division is defined.

---

## 3.  Hoare‚ÄĎlogic toolbox we will use  

| Rule | Shape | Meaning |
|------|-------|----------|
| **Assignment** | `{ Q[x‚ÜźE] }  x := E  { Q }` | Substituting `E` for `x` in the post‚ÄĎcondition. |
| **Sequencing** | `{P} S‚āĀ {R} , {R} S‚āā {Q}  ‚áí  {P} S‚āĀ;S‚āā {Q}` | Two statements executed one after the other. |
| **Input** (read) | `{P ‚ąß A}`  `x := input()`  `{‚ąÉv . (x = v) ‚ąß P ‚ąß A(v)}` | The variable receives an arbitrary value that satisfies the *input‚ÄĎassertion* `A`. |
| **Output** (print) | `{P ‚ąß B}`  `print(e)`  `{P ‚ąß B ‚ąß (output = ‚ü¶e‚üß)}` | If the *output‚ÄĎassertion* `B` holds before printing, after printing we additionally know that the *observable* output is the value of the expression `e`. |
| **Consequence** | From `{P} S {Q}` we may infer `{P' } S {Q'}` if `P' ‚áí P` and `Q ‚áí Q'`. | Strengthening the pre‚ÄĎcondition or weakening the post‚ÄĎcondition. |

In the proof we will also need a **semantic lemma** about the Python format string `"{:.2f}".format(v)`.  Let  

```
fmt2(v)  ‚ČĚ  the decimal string produced by Python when executing
           "{:.2f}".format(v)
```

The lemma (proved separately from the language definition) states:

> **Lemma (Formatting)**  
> For every real number `v` the string `fmt2(v)` is the unique decimal
> representation with exactly two digits after the decimal point that
> results from rounding `v` to the nearest hundredth (ties to even, as
> Python does).  

We will treat `fmt2(v)` as a pure mathematical function in the post‚ÄĎcondition.

---

## 4.  Decomposition of the program  

The program consists of two *atomic* statements (from the point of view of Hoare logic):

1. **Read‚ÄĎand‚ÄĎassign**  

   ```python
   Distancia, Diametro1, Diametro2 = map(int, input().split(' '))
   ```

   We model this as three consecutive assignments after a single input
   operation that yields a *triple* of integer values.

2. **Print**  

   ```python
   print("{:.2f}".format(Distancia/(Diametro1 + Diametro2)))
   ```

   This is a single output statement whose argument is the arithmetic
   expression `Distancia/(Diametro1+Diametro2)` wrapped by the formatting
   function `fmt2`.

Thus the program can be written, in an abstract Hoare‚ÄĎlogic language, as  

```
S‚āĀ ; S‚āā
```

with  

```
S‚āĀ :  (Distancia, Diametro1, Diametro2) := read_int_triplet()
S‚āā :  output := fmt2( Distancia / (Diametro1 + Diametro2) )
```

where `read_int_triplet()` is a primitive that returns a triple
`(d,a,b)` satisfying `d, a, b ‚ąą ‚Ą§`.

---

## 5.  Hoare triples for the individual statements  

### 5.1  Triple for the input statement `S‚āĀ`

**Pre‚ÄĎcondition for `S‚āĀ`**  
We need the same condition that appears in the global pre‚ÄĎcondition, but *without* the division‚ÄĎby‚ÄĎzero guard (because we have not yet examined the sum of the diameters).  Hence we take  

```
P‚āĀ ‚Č° true   (no restriction yet, the input primitive itself will impose
             the ‚Äúthree integers‚ÄĚ requirement)
```

**Post‚ÄĎcondition for `S‚āĀ`**  

After the read we know that there exist three integers `d,a,b` such that

```
Distancia = d ‚ąß Diametro1 = a ‚ąß Diametro2 = b ‚ąß a + b ‚Č† 0
```

The last conjunct (`a+b‚Č†0`) is **not** guaranteed by the input primitive,
so we keep it as an *assumption* that will be required later.  Formally

```
R‚āĀ ‚Č° ‚ąÉ d,a,b ‚ąą ‚Ą§ . (Distancia = d ‚ąß Diametro1 = a ‚ąß Diametro2 = b ‚ąß a + b ‚Č† 0)
```

Using the **Input** rule we obtain the Hoare triple

```
{ P‚āĀ }  S‚āĀ  { R‚āĀ }
```

*Proof sketch*: the primitive `read_int_triplet()` yields a triple of
integers; we bind them to the three variables; the existential quantifier
captures the fact that the concrete values are unknown but exist.

### 5.2  Triple for the output statement `S‚āā`

We now need a pre‚ÄĎcondition that guarantees that the denominator is not
zero, because the expression `Distancia/(Diametro1+Diametro2)` must be
well‚ÄĎdefined.  From `R‚āĀ` we can extract exactly this information:

```
P‚āā ‚Č° ‚ąÉ d,a,b ‚ąą ‚Ą§ . (Distancia = d ‚ąß Diametro1 = a ‚ąß Diametro2 = b ‚ąß a + b ‚Č† 0)
```

(Notice that `P‚āā` is *identical* to `R‚āĀ`; we will simply reuse it.)

**Expression evaluated in the print**  

```
e ‚Č° fmt2( Distancia / (Diametro1 + Diametro2) )
```

**Post‚ÄĎcondition after the print**  

The only observable effect of `S‚āā` is that the *output channel* receives
the string `e`.  Thus we define  

```
Q‚āā ‚Č° (output = fmt2( Distancia / (Diametro1 + Diametro2) ))
```

Applying the **Output** rule we get

```
{ P‚āā }  S‚āā  { P‚āā ‚ąß Q‚āā }
```

The rule adds the fact that the output equals the value of the expression,
while the rest of the state (`P‚āā`) is unchanged.

---

## 6.  Composition of the two triples  

We have  

```
{ P‚āĀ }   S‚āĀ   { R‚āĀ }                (1)
{ P‚āā }   S‚āā   { P‚āā ‚ąß Q‚āā }           (2)
```

and we know that `R‚āĀ` **implies** `P‚āā` (they are the same formula).  
Therefore we can apply the **Sequencing** rule:

```
{ P‚āĀ }   S‚āĀ ; S‚āā   { P‚āā ‚ąß Q‚āā }
```

Since `P‚āĀ` is `true`, the pre‚ÄĎcondition of the whole program reduces to the
global pre‚ÄĎcondition we wanted, namely

```
P ‚Č° (input consists of three integers d,a,b) ‚ąß (a + b ‚Č† 0)
```

Indeed `P‚āĀ ‚áí P` holds because `P‚āĀ` is `true` and `P` is a property of the
*environment* (the user‚Äôs input).  By the **Consequence** rule we may
strengthen the pre‚ÄĎcondition from `true` to `P`:

```
{ P }   S‚āĀ ; S‚āā   { P‚āā ‚ąß Q‚āā }          (3)
```

Now we examine the post‚ÄĎcondition `P‚āā ‚ąß Q‚āā`.  Unfolding the existential
quantifier we obtain

```
‚ąÉ d,a,b ‚ąą ‚Ą§ .
   (Distancia = d ‚ąß Diametro1 = a ‚ąß Diametro2 = b ‚ąß a + b ‚Č† 0
    ‚ąß output = fmt2( d / (a + b) ))
```

Because the three variables are *exactly* the values that were read from
the input, the existential quantifier can be eliminated (we can replace
`d,a,b` by the current values of the program variables).  Consequently we
derive the **desired post‚ÄĎcondition** `Q`:

```
Q ‚Č° output = fmt2( Distancia / (Diametro1 + Diametro2) )
```

and the side condition `Diametro1 + Diametro2 ‚Č† 0` is already part of the
global pre‚ÄĎcondition `P`.  Hence, by another application of the
**Consequence** rule (weakening the post‚ÄĎcondition), from (3) we obtain

```
{ P }   program   { Q }
```

which is exactly the Hoare triple required by the specification.

## 7.  Summary of the formal proof  

| Step | Hoare triple | Reason |
|------|--------------|--------|
| 1 | `{ true }  S‚āĀ  { ‚ąÉd,a,b‚ąą‚Ą§ . (Distancia=d ‚ąß Diametro1=a ‚ąß Diametro2=b ‚ąß a+b‚Č†0) }` | Input rule |
| 2 | `{ ‚ąÉd,a,b‚ąą‚Ą§ . (Distancia=d ‚ąß Diametro1=a ‚ąß Diametro2=b ‚ąß a+b‚Č†0) }  S‚āā  { ‚Ä¶ ‚ąß output = fmt2(Distancia/(Diametro1+Diametro2)) }` | Output rule |
| 3 | `{ true }  S‚āĀ;S‚āā  { ‚Ä¶ ‚ąß output = fmt2(Distancia/(Diametro1+Diametro2)) }` | Sequencing (1)+(2) |
| 4 | `{ P }  S‚āĀ;S‚āā  { ‚Ä¶ ‚ąß output = fmt2(Distancia/(Diametro1+Diametro2)) }` | Consequence (strengthen pre‚ÄĎcondition) |
| 5 | `{ P }  program  { output = fmt2(Distancia/(Diametro1+Diametro2)) }` | Consequence (weaken post‚ÄĎcondition, eliminate the existential) |

Thus the program is **correct** with respect to the Hoare specification:

*If the input consists of three integers `Distancia`, `Diametro1`,
`Diametro2` such that `Diametro1 + Diametro2 ‚Č† 0`, then the program
terminates and prints the value `Distancia/(Diametro1+Diametro2)` rounded
to two decimal places.*

## 8.  Remarks on the assumptions we made  

1. **Division by zero** ‚Äď The pre‚ÄĎcondition explicitly forbids the case
   `Diametro1 + Diametro2 = 0`.  In the original problem statement the
   test data never contains this situation, but from a formal point of
   view it is essential to state it.

2. **Formatting function** ‚Äď We treated `"{:.2f}".format(v)` as a pure
   mathematical function `fmt2(v)`.  The lemma about `fmt2` guarantees
   that the string produced is exactly the one required by the problem
   specification (two decimal digits, correct rounding).  No further
   reasoning about the internal implementation of Python‚Äôs formatting is
   needed for the Hoare proof.

3. **Termination** ‚Äď Both statements are *deterministic* and have no
   loops or recursion, so termination follows trivially.  In Hoare logic
   a terminating program is assumed unless a `while`/`for` construct is
   present.

## 9.  Concluding statement  

With the pre‚ÄĎcondition  

```
P ‚Č° (input is three integers d,a,b) ‚ąß (a + b ‚Č† 0)
```

and the post‚ÄĎcondition  

```
Q ‚Č° (the program prints fmt2(d/(a+b)))
```

we have derived, using the standard Hoare‚ÄĎlogic inference rules, the
triple `{P} program {Q}`.  Hence the program is **correct** with respect
to the given specification. ‚ąé
