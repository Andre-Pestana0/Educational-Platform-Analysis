## 1.  Specification  

We first make the intended pre‚ÄĎ and post‚ÄĎconditions explicit.

*Let*  

* `I = ‚ü®i‚āĀ , i‚āā , ‚Ä¶ , i‚āô‚ü©` be the (finite) sequence of input lines that the user will type.  
* Each line `i‚āĖ` is either  

  * the literal string **`caw caw`** (a *scream*), or  
  * a non‚ÄĎempty string over the alphabet `{ '*', '.' }` (the *eye pattern*).  

* The input is **well‚ÄĎformed**: it consists of three blocks, each block being a (possibly empty) eye‚ÄĎpattern line followed by a scream line.  
  Formally  

  ```
  I =  (e‚āĀ  caw caw) (e‚āā  caw caw) (e‚āÉ  caw caw)
  ```

  where each `e‚Īľ ‚ąą { '*', '.' }*` (the empty string is allowed).

*For a given eye‚ÄĎpattern `e = a‚āĀ a‚āā ‚Ä¶ a_m` we define its **value** `val(e)` as*

```
val(e) = ő£_{k = 1..m}  ( 4 // 2^{k-1} ) ¬∑ 1_{a_k = '*'}
```

i.e. the left‚ÄĎmost eye contributes 4, the next 2, the next 1, ‚Ä¶ (integer division).

*Pre‚ÄĎcondition*  

```
{  I is a well‚ÄĎformed input as described above  }
```

*Post‚ÄĎcondition*  

```
{  The program terminates after having printed, in this order,
   val(e‚āĀ), val(e‚āā), val(e‚āÉ)   (one integer per line)   }
```

## 2.  Program (P)

```python
grito_cont = 0                # (1)
valor_previsto = 0            # (2)

while grito_cont < 3:         # (3)
    entrada = input()        # (4)

    if entrada == 'caw caw': # (5)
        print(valor_previsto)# (6)
        grito_cont += 1      # (7)
        valor_previsto = 0  # (8)
    else:                     # (9)
        valor_olho = 4        # (10)
        for olho in entrada: # (11)
            if olho == '*':  # (12)
                valor_previsto += valor_olho   # (13)
            valor_olho //= 2                 # (14)
```

The only mutable program variables are  

```
grito_cont  ‚ąą ‚Ąē          # number of screams already processed
valor_previsto ‚ąą ‚Ąē       # accumulated value of the current eye‚ÄĎpattern
valor_olho ‚ąą ‚Ąē           # weight of the currently examined eye (4,2,1,‚Ä¶)
```

## 3.  Loop Invariants  

### 3.1  Outer `while` loop invariant `I‚āĀ`

```
I‚āĀ  ‚Čú  (0 ‚Č§ grito_cont ‚Č§ 3)  ‚ąß
      (‚ąÄj < grito_cont . printed_j = val(e_j))  ‚ąß
      (grito_cont < 3 ‚áí  valor_previsto = val(e_{grito_cont+1, partial}))
```

*Explanation*  

* `grito_cont` counts how many scream lines have already been **consumed**.  
* For each already‚ÄĎconsumed block `j` we have already printed the correct value `val(e_j)`.  
* If we have not yet reached three screams (`grito_cont < 3`), then `valor_previsto` holds the value of the **eye‚ÄĎpattern that is currently being read** ‚Äď possibly only a *partial* prefix of it if we are still inside the inner `for` loop (the invariant is later refined for that inner loop).

### 3.2  Inner `for` loop invariant `I‚āā`

Inside the `else`‚ÄĎbranch we execute

```python
valor_olho = 4
for olho in entrada:
    ‚Ä¶
```

For a concrete execution of the `else`‚ÄĎbranch we denote the eye‚ÄĎpattern read by `entrada` as `e = a‚āĀ a‚āā ‚Ä¶ a_m`.  
Let `p` be the number of iterations already performed (i.e. we have examined `a‚āĀ ‚Ä¶ a_p`).  

The invariant for the `for` loop is

```
I‚āā(p)  ‚Čú  valor_previsto = ő£_{k = 1..p} (4 // 2^{k-1})¬∑1_{a_k = '*'}
          ‚ąß
          valor_olho   = 4 // 2^{p}
```

When the loop finishes (`p = m`) we obtain exactly `valor_previsto = val(e)` and `valor_olho = 4 // 2^{m}` (the latter value is irrelevant after the loop).

## 4.  Proof Structure  

We prove the Hoare triples  

```
{ Pre }   P   { Post }
```

by the usual rule: establish the invariant at loop entry, prove preservation, and prove that the invariant together with the loop exit condition implies the post‚ÄĎcondition.

### 4.1  Initialization of the outer loop

*After statements (1)‚Äď(2)*  

```
{ true }
grito_cont = 0;          // (1)
valor_previsto = 0;      // (2)
{ I‚āĀ }
```

*Proof*:  

* `0 ‚Č§ grito_cont ‚Č§ 3` holds because `grito_cont = 0`.  
* No scream has been processed, therefore the universal quantifier in `I‚āĀ` is vacuously true.  
* Because no eye‚ÄĎpattern has been read yet, the ‚Äúpartial‚ÄĚ value is `0`, which equals `valor_previsto`. ‚ąé

### 4.2  Preservation of `I‚āĀ` by the body of the `while`

We consider the two branches separately.

#### 4.2.1  The **scream** branch (`entrada == 'caw caw'`)

Assume the invariant `I‚āĀ` holds at the beginning of the iteration and that the test succeeds.

1. **Printing** (`print(valor_previsto)`) ‚Äď by the definition of `I‚āĀ` (case `grito_cont < 3`) we have `valor_previsto = val(e_{grito_cont+1})`. Hence the printed number is exactly the required value for the current block.

2. **Update `grito_cont`** (`grito_cont += 1`) ‚Äď the counter is increased from `k` to `k+1`.  

   After this statement the first conjunct of `I‚āĀ` (`0 ‚Č§ ‚Ä¶ ‚Č§ 3`) still holds because `k < 3` (the loop guard) implies `k+1 ‚Č§ 3`.

3. **Reset `valor_previsto`** (`valor_previsto = 0`) ‚Äď now the ‚Äúpartial value‚ÄĚ for the *next* eye‚ÄĎpattern is correctly set to 0, which is the value of the empty prefix of the (as yet unread) pattern `e_{k+2}`.  

Thus after the three statements we obtain `I‚āĀ` again, with `grito_cont` increased by one and `valor_previsto` reset.

#### 4.2.2  The **eye‚ÄĎpattern** branch (`entrada != 'caw caw'`)

Assume `I‚āĀ` holds and the test fails.  
Let `entrada = e = a‚āĀ a‚āā ‚Ä¶ a_m`.  
We must show that after the execution of the `for` loop we have

```
valor_previsto = val(e)            (new partial value)
valor_olho   = 4 // 2^{m}           (irrelevant afterwards)
```

*Initialization of the inner loop* (`valor_olho = 4`) establishes `I‚āā(0)` because the empty sum is 0 and `4 // 2^{0} = 4`.

*Preservation of `I‚āā`* ‚Äď consider one iteration where the loop variable `olho` is the next character `a_{p+1}`.

*Case `a_{p+1} == '*'`*  

```
valor_previsto := valor_previsto + valor_olho
valor_olho    := valor_olho // 2
```

By the induction hypothesis

```
valor_previsto = ő£_{k=1..p} (4 // 2^{k-1})¬∑1_{a_k='*'}
valor_olho    = 4 // 2^{p}
```

Hence after the addition we obtain  

```
valor_previsto' = ő£_{k=1..p} ‚Ä¶  + (4 // 2^{p})¬∑1_{a_{p+1}='*'}
                = ő£_{k=1..p+1} (4 // 2^{k-1})¬∑1_{a_k='*'}
```

and after the division  

```
valor_olho' = (4 // 2^{p}) // 2 = 4 // 2^{p+1}
```

which is precisely `I‚āā(p+1)`.

*Case `a_{p+1} == '.'`*  

The body does **not** modify `valor_previsto`; the same algebraic reasoning shows that the invariant still holds because the added term is zero.  

Thus `I‚āā` is preserved.

*Termination of the inner loop* ‚Äď when the loop has processed all `m` characters we have `I‚āā(m)`, i.e.

```
valor_previsto = ő£_{k=1..m} (4 // 2^{k-1})¬∑1_{a_k='*'} = val(e)
valor_olho   = 4 // 2^{m}
```

No further statements in the `else`‚ÄĎbranch touch `valor_previsto`, therefore the outer invariant `I‚āĀ` is restored with the same `grito_cont` (no scream has been counted) and with the updated partial value `valor_previsto = val(e)`.

Consequently **both branches** preserve `I‚āĀ`.

### 4.3  Exit of the outer loop

The loop guard is `grito_cont < 3`.  
When the loop terminates we have `¬¨(grito_cont < 3)`, i.e. `grito_cont = 3`.  
From `I‚āĀ` we obtain:

* All three scream blocks have been processed (`grito_cont = 3`).  
* For each `j = 1,2,3` the statement ‚Äúthe *j‚ÄĎth* printed number equals `val(e_j)`‚ÄĚ holds (the universal conjunct of `I‚āĀ`).  

Thus the post‚ÄĎcondition follows directly.

## 5.  Formal Hoare triples

Below we list the essential triples, using the notation `{P} S {Q}` for a Hoare triple.

1. **Initialisation**

   ```
   { true }
   grito_cont = 0; valor_previsto = 0;
   { I‚āĀ }
   ```

2. **Outer while (rule of iteration)**  

   Assuming `{ I‚āĀ ‚ąß grito_cont < 3 } body { I‚āĀ }` we get  

   ```
   { I‚āĀ } while grito_cont < 3 do body od { I‚āĀ ‚ąß grito_cont ‚Č• 3 }
   ```

3. **Body ‚Äď scream branch**

   ```
   { I‚āĀ ‚ąß entrada = 'caw caw' }
   print(valor_previsto); 
   grito_cont := grito_cont + 1; 
   valor_previsto := 0;
   { I‚āĀ }
   ```

4. **Body ‚Äď eye‚ÄĎpattern branch (inner loop)**  

   *Inner loop invariant* `I‚āā(p)` is proved by the standard **for**‚ÄĎrule (or by converting the `for` into a `while`).  

   ```
   { I‚āĀ ‚ąß entrada = e ‚Č† 'caw caw' }
   valor_olho := 4;
   while (more characters) do
        (if olho = '*' then valor_previsto := valor_previsto + valor_olho);
        valor_olho := valor_olho // 2
   od
   { I‚āĀ }          // because after the inner loop valor_previsto = val(e)
   ```

5. **Termination**

   From the exit condition `grito_cont = 3` and `I‚āĀ` we derive the post‚ÄĎcondition.

All proof obligations are discharged, therefore

```
{ Pre }   P   { Post }
```

holds. The program is **partially correct** with respect to the given specification.  

Because the loop guard `grito_cont < 3` is a linear integer expression that strictly increases (`grito_cont` is incremented exactly when a scream line is read) and the input is assumed finite, the program also **terminates**, giving **total correctness**.

## 6.  Intuitive interpretation  

* The variable `valor_olho` plays the role of the weight `4,2,1,‚Ä¶`.  
* The inner `for` loop computes the binary value of the eye pattern, exactly the definition of `val(e)`.  
* Each time a scream (`caw caw`) appears the accumulated value is printed, the counter of screams is increased, and the accumulator is cleared for the next block.  
* After three screams the outer loop stops, guaranteeing that exactly three numbers have been printed, and each of them is the value of the eye pattern that immediately preceded the corresponding scream.

Thus the program fulfills the problem requirement, and the Hoare‚ÄĎlogic proof above shows it does so **by construction**. ‚ąé
