**Answer ‚Äď a Hoare‚ÄĎlogic proof that the program satisfies its specification**

Below we give a complete Hoare‚ÄĎlogic proof that the program posted in the question
computes, for a given album, the number of stickers that are still missing after
the user has bought a certain number of stickers.  
The proof is written in the usual *Hoare triple* form  

\[
\{P\}\;C\;\{Q\}
\]

where  

* **\(P\)** ‚Äď the *pre‚ÄĎcondition* (what we know before the program starts),  
* **\(C\)** ‚Äď the program (a sequence of commands),  
* **\(Q\)** ‚Äď the *post‚ÄĎcondition* (what we must guarantee after the program terminates).

The proof proceeds by (i) defining a suitable loop invariant for the `for`‚ÄĎloop,
(ii) showing that the invariant holds initially, (iii) that it is preserved by
each iteration, (iv) that it yields the desired post‚ÄĎcondition when the loop
terminates, and (v) that the loop always terminates (so the program is total).

---

## 1.  Specification

### 1.1  Variables

| Symbol | Meaning (at program start) |
|--------|----------------------------|
| `total_fig` | total number of different stickers that exist in the album (‚Č•‚ÄĮ0) |
| `total_figCompradas` | number of stickers that were bought (‚Č•‚ÄĮ0) |
| `fig` | the identifier of a sticker that is read in the current iteration |
| `fig_possuidas` | list that will contain **distinct** stickers already seen |
| `n` | the initial value of `total_fig` (a constant) |
| `m` | the initial value of `total_figCompradas` (a constant) |
| `D(k)` | set of distinct stickers among the first `k` inputs (0‚ÄĮ‚Č§‚ÄĮk‚ÄĮ‚Č§‚ÄĮm) |

### 1.2  Pre‚ÄĎcondition  

\[
P\;:\; n\ge 0 \;\land\; m\ge 0 \;\land\; \text{the next }m\text{ inputs are integers in }[1..n]
\]

(We do not need to know the concrete values of the stickers; only that they are
valid identifiers.)

### 1.3  Post‚ÄĎcondition  

After the program finishes it prints  

\[
Q\;:\; \text{output}=n-|D(m)|
\]

i.e. the initial number of stickers **minus** the number of *different* stickers
that were bought.  Because `|D(m)|‚Č§n`, the result is always a non‚ÄĎnegative
integer.

---

## 2.  Program (annotated)

```python
total_fig = int(input())          # line 1
total_figCompradas = int(input()) # line 2

fig_possuidas = []                # line 3

for i in range(total_figCompradas):   # line 4  (loop)
    fig = int(input())                 # line 5
    if(fig_possuidas.count(fig)==0):   # line 6
        fig_possuidas.append(fig)      # line 7
        total_fig -= 1                 # line 8

print(total_fig)                  # line 9
```

We will treat the `for`‚ÄĎloop as a *while*‚ÄĎloop with a counter `i` that runs from
`0` to `total_figCompradas‚ÄĎ1`.  The body of the loop is the block `B` consisting
of lines‚ÄĮ5‚ÄĎ8.

---

## 3.  Loop invariant

Let  

\[
\mathcal I(i) \;:\;
\begin{cases}
0\le i\le m,\\[2mm]
\text{`fig_possuidas` contains exactly the set } D(i),\\[2mm]
\text{`total_fig` = } n - |D(i)|.
\end{cases}
\]

In words: after we have processed the first `i` stickers, the list `fig_possuidas`
holds *all* distinct stickers seen so far (no duplicates), and `total_fig` has
been decreased precisely once for each distinct sticker.

The invariant is strong enough to give us the post‚ÄĎcondition when the loop ends
(`i = m`), and it is easy to prove that it is preserved.

---

## 4.  Proof steps

### 4.1  Initialization (before the loop)

At the moment just before the loop (`i = 0`):

* `fig_possuidas` is `[]`. Hence it contains the empty set `D(0) = ‚ąÖ`.
* `total_fig` still holds the original value `n`.  
  Since `|D(0)| = 0`, we have `total_fig = n - 0 = n`.

Therefore `\mathcal I(0)` holds.

```
{ P }
total_fig = n; total_figCompradas = m; fig_possuidas = []
{ \mathcal I(0) }
```

### 4.2  Maintenance (preservation of the invariant)

Assume `\mathcal I(i)` holds at the beginning of an iteration, where `0 ‚Č§ i < m`.
We must show that after executing the body `B` we obtain `\mathcal I(i+1)`.

The body reads a new sticker `fig`. Two cases are possible.

| Case | Condition | Effect of the body | New set `D(i+1)` | New value of `total_fig` |
|------|-----------|-------------------|-----------------|--------------------------|
| **A** | `fig` **already** in `fig_possuidas` (`count(fig) ‚Č† 0`) | No `append`, no `total_fig -= 1` | `D(i+1) = D(i)` (no new distinct sticker) | `total_fig` unchanged = `n - |D(i)| = n - |D(i+1)|` |
| **B** | `fig` **not** in `fig_possuidas` (`count(fig) = 0`) | `append(fig)`, `total_fig -= 1` | `D(i+1) = D(i) ‚ą™ {fig}` (one new distinct sticker) | `total_fig` becomes `n - |D(i)| - 1 = n - |D(i+1)|` |

In both cases the list `fig_possuidas` ends up containing exactly `D(i+1)` and
`total_fig` equals `n - |D(i+1)|`. Hence `\mathcal I(i+1)` holds.

Formally:

```
{ \mathcal I(i) ‚ąß 0 ‚Č§ i < m }
    fig = input()
    if fig_possuidas.count(fig)==0 then
        fig_possuidas.append(fig);
        total_fig := total_fig - 1
    fi
{ \mathcal I(i+1) }
```

Thus the invariant is preserved.

### 4.3  Termination (loop exit)

The loop counter `i` is increased by one at each iteration (`i := i+1` inside the
`for`). Because the loop condition is `i < m` and `m` is a non‚ÄĎnegative integer,
the loop can execute at most `m` times. Therefore it **terminates** after
exactly `m` iterations, reaching the state `i = m`.

### 4.4  Post‚ÄĎcondition

When the loop finishes we have `i = m`. By the invariant `\mathcal I(m)` we know

* `fig_possuidas` contains the set `D(m)` of *all* distinct stickers that were
  read,
* `total_fig = n - |D(m)|`.

The final statement `print(total_fig)` therefore outputs `n - |D(m)|`, which is
exactly the required result.

```
{ \mathcal I(m) }
print(total_fig)
{ output = n - |D(m)| }   (which is Q)
```

---

## 5.  Full Hoare triple

Putting the pieces together we obtain the total Hoare triple

\[
\boxed{
\{\,P\,\}\;
\text{program}
\;\{\,Q\,\}
}
\]

where  

* `P` is the pre‚ÄĎcondition defined in ¬ß‚ÄĮ1.2,  
* `Q` is the post‚ÄĎcondition defined in ¬ß‚ÄĮ1.3,  
* the proof consists of the three steps *initialisation*, *maintenance*,
  *termination* shown in ¬ß‚ÄĮ4.

Hence the program is **correct** with respect to the specification: it always
produces the number of missing stickers after processing the list of purchased
stickers.

---

## 6.  Remarks on the proof style

* The invariant `\mathcal I(i)` is *exactly* the property that the program is
  maintaining ‚Äď it captures both the content of the auxiliary list and the
  relationship between `total_fig` and the number of distinct stickers seen.
* The proof uses only elementary arithmetic (addition/subtraction) and set
  cardinalities; no external libraries or side‚ÄĎeffects are needed.
* Because the loop variable `i` is bounded by the constant `m`, the proof of
  termination is immediate ‚Äď this satisfies the total‚ÄĎcorrectness requirement
  of Hoare logic (partial correctness + termination).
