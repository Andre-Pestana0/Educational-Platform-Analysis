## 1.  What we have to prove  

We are given a small Python program that reads  

* an integer `n` – the “correct” tea number;  
* a (non‑empty) line containing a sequence of integers `respostas` – the guesses;  

and finally prints the number `acertos` of guesses that are equal to `n`.

Formally we want to prove  

\[
\{\,P\,\}\; \text{program}\; \{\,Q\,\}
\]

where  

* **Pre‑condition  \(P\)**  

  \[
  P \equiv
  \begin{cases}
   n\in\mathbb Z,\\[2mm]
   respostas = \langle r_0,\dots ,r_{k-1}\rangle\quad\text{with }k\ge 0\text{ and } r_i\in\mathbb Z\;(\forall i<k)
  \end{cases}
  \]

* **Post‑condition  \(Q\)**  

  \[
  Q \equiv 
  acertos = |\{\,i\mid 0\le i<k\land r_i=n\,\}|
  \]

In words: after the program finishes, the variable `acertos` must contain exactly the cardinality of the set of positions whose value equals `n`.

We shall prove **partial correctness** (the program, if it terminates, yields a state satisfying \(Q\)) and then argue that the program **terminates** (total correctness).

---

## 2.  Annotated program  

Below the program is written line‑by‑line together with the assertions that we will use in a Hoare‑logic proof.  
The syntax is Python‑like, but the proof works for any imperative language with the usual assignment, input, output and `for`‑loop constructs.

```
1  n = int(input())                     { A1 }
2  respostas = list(map(int, input().split(' ')))   { A2 }
3  acertos = 0                          { A3 }
4  i = 0                                { A4 }   # loop counter (implicit in Python’s for)
5  while i < len(respostas):            { I }    # loop invariant, see below
6      if int(respostas[i]) == n:       { I ∧ i<k }
7          acertos = acertos + 1
8      i = i + 1
9  print(acertos)                       { Q }
```

*Line 1* and *Line 2* are the “input” statements.  In Hoare logic they are treated as *assume* statements that bring the variables into the current state in accordance with the pre‑condition.  
The rest of the proof concentrates on the computation of `acertos`.

---

## 3.  Loop invariant  

The crucial part of a Hoare‑logic proof for a loop is an **invariant** that is true

* before the first iteration,
* preserved by one iteration, and
* strong enough to imply the post‑condition when the loop finishes.

We choose  

\[
\boxed{I\; \equiv\;
   0\le i\le k\;\land\;
   acertos = |\{\,j\mid 0\le j < i \land r_j=n\,\}|
   }
\]

where  

* `k = len(respostas)`,
* `r_j` denotes the `j`‑th element of the list `respostas`.

Intuitively, after having examined the first `i` elements, `acertos` stores exactly the number of matches among those `i` elements.

---

## 4.  Proof steps  

### 4.1  Initialization  

We have to show  

\[
\{\,P\,\}\;\text{Lines 1–4}\;\{\,I\,\}
\]

* After *Line 1* and *Line 2* the variables `n` and `respostas` satisfy the part of \(P\) that talks about them.  
* *Line 3* sets `acertos` to 0.  
* *Line 4* sets `i` to 0.  

Now  

* `0 ≤ i = 0 ≤ k` (because `k = len(respostas)` is a non‑negative integer), and  
* the set `{ j | 0 ≤ j < 0 ∧ r_j = n }` is empty, so its cardinality is 0, which equals `acertos`.  

Hence `I` holds.

---

### 4.2  Preservation (the body)  

We must prove the Hoare triple for the loop body under the invariant and the guard:

\[
\{\, I \land i<k \,\}\;
\text{if … then … ; i = i+1}\;
\{\, I \,\}
\]

Assume `I ∧ i<k` holds at the beginning of an iteration.

* **Case 1 – the test succeeds** (`respostas[i] == n`).  

  *Before the `if`*: by `I`, `acertos` equals the number of matches among the first `i` elements.  
  *Inside the `then`*: we execute `acertos = acertos + 1`; therefore after this statement  
  \[
  acertos = |\{\,j\mid 0\le j<i\land r_j=n\,\}| + 1 
          = |\{\,j\mid 0\le j\le i\land r_j=n\,\}|.
  \]  
  *After the `if`* (whether the branch was taken or not) we execute `i = i+1`.  
  The new value `i' = i+1` satisfies `0 ≤ i' ≤ k` (because the guard guarantees `i+1 ≤ k`).  
  Moreover the set of indices counted in the invariant becomes exactly  
  \[
  \{\,j\mid 0\le j< i' \land r_j=n\,\}
  = \{\,j\mid 0\le j\le i \land r_j=n\,\},
  \]  
  whose cardinality is precisely the current value of `acertos`.  
  Hence `I` holds with the updated `i`.

* **Case 2 – the test fails** (`respostas[i] != n`).  

  The `then` branch is skipped, leaving `acertos` unchanged.  
  Since the element `r_i` is **not** a match, the set of matches among the first `i+1` elements is the same as the set among the first `i` elements, i.e. its cardinality is still `acertos`.  
  After `i = i+1` the same reasoning as in Case 1 shows that `I` holds again.

Thus in every possible execution of the body, `I` is preserved.

---

### 4.3  Termination  

The loop condition is `i < k`.  
Inside the body we always execute `i = i + 1`.  
Since `i` is a non‑negative integer bounded above by `k`, after at most `k` iterations the condition becomes false and the loop exits.  
Therefore the loop **terminates** (total correctness).

---

### 4.4  Post‑condition  

When the loop ends we have `¬(i < k)`, i.e. `i = k`.  
By the invariant `I` (which is still true) we obtain  

\[
acertos = |\{\,j\mid 0\le j < i \land r_j=n\,\}|
        = |\{\,j\mid 0\le j < k \land r_j=n\,\}|.
\]

But `k = len(respostas)`, so the right‑hand side is exactly the number of guesses equal to `n`.  
Hence the post‑condition `Q` holds.

The final statement `print(acertos)` does not modify any variable, therefore it preserves `Q`.  

Thus we have proved  

\[
\{\,P\,\}\; \text{program}\; \{\,Q\,\}
\]

and, together with the termination argument, the program is **totally correct** with respect to the specification.

---

## 5.  Summary of the Hoare proof  

| Line | Hoare triple (partial correctness) |
|------|--------------------------------------|
| 1–2  | `\{P\}  …  \{ n∈ℤ ∧ respostas = ⟨r₀,…,r_{k‑1}⟩ \}` |
| 3    | `\{…\}  acertos := 0  \{ acertos = 0 \}` |
| 4    | `\{…\}  i := 0        \{ I \}` (initialisation) |
| 5–8  | `\{ I ∧ i<k \}  body  \{ I \}` (preservation) |
| 9    | `\{ I ∧ i = k \}  print(acertos)  \{ Q \}` |

Together with the loop‑termination argument we have established **total correctness**.

---

## 6.  What this means for the original problem  

*The program reads the correct tea number `n` and a list of guesses `respostas`.  
It counts, in `acertos`, exactly how many guesses equal `n` and prints that count.*  

The Hoare‑logic proof above shows that, under the obvious assumptions on the input (integers, finite list), the program always terminates and always produces the required answer. Hence the program conforms to its formal specification.
