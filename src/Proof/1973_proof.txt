**Answer – a Hoare‑logic proof of correctness**

Below we give a *complete* correctness proof (partial‑correctness) for the
Python program that solves the “Beecrowd – Iniciante – Nível 7’’ problem.
The proof is written in the style of Hoare logic, i.e. we exhibit a
pre‑condition **P**, a post‑condition **Q**, a loop invariant **I**, and we
show the three classic obligations:

1. **Initialization** – `⊢ {P}  init  {I}`  
2. **Maintenance** – `⊢ {I ∧ B}  body  {I}`  (where `B` is the loop guard)  
3. **Termination** – `⊢ {I ∧ ¬B}  after‑loop  {Q}`  

All statements are treated as *atomic* assignments, conditionals and the
`for`‑loop.  The proof does **not** use any particular property of Python
other than the usual evaluation order of expressions.

---------------------------------------------------------------------

## 1.  Specification

### 1.1  Variables that appear in the program  

| Symbol                     | Meaning (English)                                 |
|----------------------------|---------------------------------------------------|
| `N`                        | number of stars (input)                           |
| `carneiros`                | list `a[0] … a[N‑1]` – number of sheep on each star|
| `total_carneiros`          | total number of sheep that *remain* after the thief|
| `qtd_estrelas_atacadas`    | number of stars the thief *attacks*               |
| `estrelas_com_um_carneiro` | among the already visited stars, how many started with exactly one sheep |
| `primeira_qtd_par_encontrada` | “has the first even‑sheep star already been seen?” |
| `index` (loop variable)   | index of the current star (0‑based)               |
| `qtd_carneiros_estrela`    | `a[index]` – number of sheep on the current star  |

All variables are integer‑valued.

### 1.2  Pre‑condition **P**

```
P  ≡   N ≥ 0 ∧
       len(carneiros) = N ∧
       (∀ i ∈ [0,N) : a[i] ≥ 1)               (* every star has at least one sheep *)
```

No other hypothesis is needed – the program reads exactly these values.

### 1.3  Post‑condition **Q**

Define two auxiliary quantities that are *pure* (they do not depend on
the program state, only on the input list `a`).

*Let*  

```
k =  min { i | 0 ≤ i < N ∧ a[i] is even }   if such i exists,
     N                                      otherwise.
```

`k` is the index of the **first even‑sheep star** (or `N` if no even star
exists).

```
parityFound = (k < N)                (* “even star found”  *)
```

```
sumAll = Σ_{i=0}^{N-1} a[i]           (* total number of sheep before any theft *)
```

```
oneBefore = | { i | 0 ≤ i < k ∧ a[i] = 1 } |   (* how many of the visited stars had exactly one sheep *)
```

```
stolen =  (2·k + 1) – oneBefore   if parityFound
          N                       otherwise
```

Then the required result is

```
Q  ≡   qtd_estrelas_atacadas = (k+1)               if parityFound
       qtd_estrelas_atacadas = N                 otherwise
   ∧   total_carneiros     = sumAll – stolen
```

That is exactly the description given in the problem statement:
*the thief walks left‑to‑right, steals one sheep from each visited star,
and, as soon as he meets a star with an even number of sheep, he turns
back and steals again from every already visited star except those that
initially had only one sheep*.  If no even star exists he never turns
back.

---------------------------------------------------------------------

## 2.  Loop invariant

The only loop is the `for index, qtd_carneiros_estrela in enumerate(carneiros):`
loop.  Let  

```
i  = index + 1          (* number of stars already processed, 0 ≤ i ≤ N *)
```

The invariant **I(i)** captures the exact state after the first `i`
iterations (i.e. after the body has been executed for the elements
`a[0] … a[i‑1]`).

```
I(i)  ≡
   0 ≤ i ≤ N                                            (range)

   total_sum_i = Σ_{j=0}^{i-1} a[j]                     (total of the processed stars)

   estrelas_com_um_carneiro
        = |{ j | 0 ≤ j < i ∧ a[j] = 1 }|                (how many of them started with 1)

   primeira_qtd_par_encontrada
        = (∃ j < i : a[j] even)                         (has an even star been seen?)

   if ¬primeira_qtd_par_encontrada then
        total_carneiros = total_sum_i                  (no theft has been subtracted yet)
        qtd_estrelas_atacadas = 0
   else
        let f = min { j | j < i ∧ a[j] even }          (the first even star)
        qtd_estrelas_atacadas = f + 1
        total_carneiros
            = total_sum_i – ((2·f + 1) – |{ j | j < f ∧ a[j] = 1 }|)
```

In words:

* the variables `total_sum_i` and `estrelas_com_um_carneiro` correctly
  count what has been seen so far,
* the flag `primeira_qtd_par_encontrada` is true exactly when an even
  star has already appeared,
* if the flag is still false we have **only** accumulated the sum of
  the visited stars (no theft yet);
* if the flag is true we have already performed the whole “out‑and‑back”
  theft that corresponds to the *first* even star `f`; afterwards the
  loop body only adds the raw values of later stars to `total_sum_i`,
  but it never changes `total_carneiros` nor `qtd_estrelas_atacadas`.

The invariant is strong enough to imply the post‑condition when the
loop finishes (`i = N`).

---------------------------------------------------------------------

## 3.  Proof obligations

### 3.1  Initialization (`i = 0`)

Before the loop the program executes

```python
estrelas_com_um_carneiro = 0
total_carneiros = qtd_estrelas_atacadas = 0
primeira_qtd_par_encontrada = False
```

From the pre‑condition we know that no element has been processed,
hence `i = 0`.  Substituting `i = 0` into the invariant:

* `total_sum_0 = Σ_{j=0}^{-1} a[j] = 0` – matches the initial value `total_carneiros = 0`.
* `estrelas_com_um_carneiro = 0` – matches the assignment.
* No even star has been seen, therefore `primeira_qtd_par_encontrada = False`,
  and `qtd_estrelas_atacadas = 0` – also matches the assignments.

Thus `{P}  init  {I(0)}` holds.

---------------------------------------------------------------------

### 3.2  Maintenance – one loop iteration

Assume `I(i)` holds at the **beginning** of an iteration where the loop
guard `index < N` (equivalently `i < N`) is true.
The body of the loop consists of the following statements (shown in the
order they are executed by Python):

```python
total_carneiros += qtd_carneiros_estrela      # (1)
if not primeira_qtd_par_encontrada:          # (2)
    if qtd_carneiros_estrela % 2 == 0:       # (2a)
        qtd_estrelas_atacadas = index + 1
        total_carneiros -= (index * 2) + 1 - estrelas_com_um_carneiro
        primeira_qtd_par_encontrada = True
    if qtd_carneiros_estrela - 1 == 0:       # (2b)
        estrelas_com_um_carneiro += 1
```

Recall that `qtd_carneiros_estrela = a[index]` and `index = i`.

We treat the two possible cases separately.

---------------------------------------------------------------------

#### Case A – `primeira_qtd_par_encontrada = False` at entry

*At entry* we have, by `I(i)`

```
total_carneiros = total_sum_i            (no theft yet)
estrelas_com_um_carneiro = # {j < i | a[j] = 1}
```

1. **Statement (1)** adds `a[i]` to `total_carneiros`.  
   Hence after (1)

   ```
   total_carneiros = total_sum_i + a[i] = total_sum_{i+1}
   ```

2. **Condition (2)** is true, so we execute the inner block.

   * **Sub‑case A1 – `a[i]` is even** (`a[i] % 2 == 0`).  
     The program executes the three assignments inside (2a).

     * `qtd_estrelas_atacadas ← i+1` – this is exactly `f+1` where `f = i`
       becomes the *first* even index.
     * `total_carneiros ← total_carneiros – ((i*2)+1 – estrelas_com_um_carneiro)`.  
       Using the value of `total_carneiros` just after (1) we obtain  

       ```
       total_carneiros
          = total_sum_{i+1} – ((2·i + 1) – estrelas_com_um_carneiro)
          = total_sum_{i+1} – ((2·f + 1) – |{j < f | a[j]=1}|)
       ```
       because `f = i` and `estrelas_com_um_carneiro` already counts the
       stars with one sheep among `0 … f‑1`.
     * `primeira_qtd_par_encontrada ← True`.

     After (2a) the invariant **I(i+1)** holds with `f = i` :

     * `total_sum_{i+1}` is correct,
     * `estrelas_com_um_carneiro` is unchanged (the later test (2b) will be
       examined next),
     * the flag is now true,
     * `qtd_estrelas_atacadas` and `total_carneiros` have the exact
       expressions required by the “even‑found’’ branch of the invariant.

   * **Sub‑case A2 – `a[i]` is odd** (`a[i] % 2 != 0`).  
     The inner `if` (2a) is skipped, only the second test (2b) is
     evaluated.

     * If `a[i] = 1` then `estrelas_com_um_carneiro ← estrelas_com_um_carneiro + 1`,
       otherwise it stays unchanged.
     * The flag stays false.

     Consequently after the whole iteration we have

     ```
     total_carneiros = total_sum_{i+1}
     estrelas_com_um_carneiro = # {j ≤ i | a[j] = 1}
     primeira_qtd_par_encontrada = False
     qtd_estrelas_atacadas = 0
     ```

     which is precisely `I(i+1)` for the *no‑even‑yet* case.

Thus, when the flag is false at entry, the body preserves the invariant.

---------------------------------------------------------------------

#### Case B – `primeira_qtd_par_encontrada = True` at entry

In this situation the *first* even star has already been found at some
index `f < i`.  By the invariant we already have

```
total_carneiros = total_sum_i – ((2·f + 1) – |{j < f | a[j]=1}|)
qtd_estrelas_atacadas = f + 1
primeira_qtd_par_encontrada = True
```

Now the body runs:

1. **Statement (1)** adds `a[i]` to `total_carneiros`.  
   Afterwards

   ```
   total_carneiros = total_sum_i + a[i] – ((2·f + 1) – …)
                  = total_sum_{i+1} – ((2·f + 1) – …)
   ```

2. The outer `if not primeira_qtd_par_encontrada` is **false**, therefore
   **none** of the inner statements (2a) or (2b) are executed.
   Consequently `estrelas_com_um_carneiro` and the flag keep their old
   values.

Hence after the iteration we obtain exactly the invariant `I(i+1)` with
the same `f`.  No new even star can change the already‑computed theft,
which matches the intended semantics of the algorithm.

---------------------------------------------------------------------

Since we have covered **all** possibilities, we have shown

```
⊢ { I(i) ∧ (i < N) }  body  { I(i+1) }
```

i.e. the invariant is *maintained*.

---------------------------------------------------------------------

### 3.3  Termination – after the loop (`i = N`)

When the loop finishes we have `i = N`.  The invariant `I(N)` states:

```
total_sum_N = Σ_{j=0}^{N-1} a[j] = sumAll
estrelas_com_um_carneiro = |{ j | a[j] = 1 }|
primeira_qtd_par_encontrada  ⇔  (∃ j < N : a[j] even)
```

Two sub‑cases arise.

* **Sub‑case 1 – an even star exists** (`primeira_qtd_par_encontrada = True`).  
  Let `f = min { j | a[j] even }`.  
  The invariant gives

  ```
  qtd_estrelas_atacadas = f + 1
  total_carneiros
        = sumAll – ((2·f + 1) – |{ j < f | a[j] = 1 }|)
  ```

  The program then executes the `if not primeira_qtd_par_encontrada:` block,
  whose guard is false, so nothing is changed.
  Consequently the final state satisfies the right‑hand side of **Q**.

* **Sub‑case 2 – no even star exists** (`primeira_qtd_par_encontrada = False`).  
  The invariant tells us that at this point

  ```
  total_carneiros = sumAll          (no theft has been subtracted yet)
  ```

  The program executes the *else* part:

  ```python
  qtd_estrelas_atacadas = N
  total_carneiros -= N
  ```

  Hence

  ```
  qtd_estrelas_atacadas = N
  total_carneiros       = sumAll – N
  ```

  which is exactly the “no‑even‑star’’ clause of **Q**.

Thus in both cases the state after the loop (and after the final `if`)
implies **Q**.  Formally:

```
⊢ { I(N) }  after‑loop  { Q }
```

---------------------------------------------------------------------

## 4.  Concluding theorem

We have proved the three Hoare triples

```
{ P }  init               { I(0) }
{ I(i) ∧ i < N } body    { I(i+1) }   for all i = 0 … N‑1
{ I(N) } after‑loop       { Q }
```

By the **Rule of Consequence** and the **Loop Rule** of Hoare logic we
obtain the total correctness triple

```
{ P }  whole_program  { Q }
```

Therefore the program is **partially correct** with respect to the
given specification: whenever it terminates (it always does, because the
`for` loop iterates a finite number `N` of times), it prints exactly the
pair

```
(qtd_estrelas_atacadas , total_carneiros)
```

required by the problem statement.

---------------------------------------------------------------------

## 5.  Remarks on the proof style

* The invariant is *exactly* the mathematical description of the thief’s
  behaviour; it is strong enough to survive the whole loop and to give
  the required result after termination.
* The proof never needed to reason about the concrete Python data‑type
  implementation (lists, integers) – only about the arithmetic
  relationships.
* The auxiliary quantities `k`, `f`, `sumAll`, `oneBefore`, `stolen`
  introduced in the specification are *derived* from the program’s
  variables; they are used only to make the post‑condition readable.
* The proof is a classic example of **loop‑invariant** reasoning: the
  flag `primeira_qtd_par_encontrada` splits the execution into two
  *phases* (before the first even star, after it).  The invariant keeps
  track of which phase we are in and of the exact amount already stolen
  in the first phase.

---------------------------------------------------------------------

**Thus the program satisfies the given pre‑ and post‑conditions, i.e.
it is correct according to the Hoare‑logic specification.**
