```
{   input  op ∈ {‘S’,‘M’}  and  a[0..11][0..11]  are 144 real numbers   }
P

The proof proceeds in the usual three‑step fashion  

1. **State the pre‑ and post‑conditions** (the specification).  
2. **Identify loop invariants** for the two nested `for`‑loops and prove that they are preserved.  
3. **Show that after the loops the variable `soma` holds exactly the required sum** and that the final `if`‑statement produces the required result.

All reasoning is written as Hoare triples  

```
{P}  C  {Q}
```

where `P` is the assertion before command `C` and `Q` the assertion after it.  
The proof uses the standard Hoare‑logic rules for *assignment*, *sequence*, *conditional* and *while*/*for* loops (the latter is treated as a `while` with an explicit loop counter).

## 1. Specification (Pre‑ and Post‑conditions)

Let  

* `op` be the character read from `input()`.  
* `a[i][j]` (`0 ≤ i,j ≤ 11`) be the real value read for the matrix entry in row `i` and column `j`.  
* `BelowSec(i,j) ≡ (j > 11‑i)` – the predicate that tells whether the element `(i,j)` lies **strictly below the secondary diagonal** (the diagonal that runs from the upper‑right corner `(0,11)` to the lower‑left corner `(11,0)`).

Define the **desired sum**

```
S  ≡  Σ_{i=0}^{11} Σ_{j=0}^{11}  [BelowSec(i,j)] · a[i][j]
```

The secondary diagonal contains exactly 12 elements; the region “below” it contains `66` elements (the triangular part of the 12×12 matrix).  

The **post‑condition** demanded by the problem is

```
Post(op, S) ≡
    (op = 'S' →  output =  S)   ∧
    (op ≠ 'S' →  output =  S / 66)
```

The **pre‑condition** simply guarantees that the input is well‑formed:

```
Pre  ≡  op ∈ {'S','M'}  ∧  ∀ i,j ∈ {0,…,11} : a[i][j] ∈ ℝ
```

Thus the whole specification is the Hoare triple  

```
{Pre}   Program   {Post(op, S)}
```

## 3. Detailed Hoare‑logic proof

### 3.1. Reading the operation

```
{Pre}
op = input()
{  op ∈ {'S','M'}  ∧  ∀i,j : a[i][j] ∈ ℝ   }
```

The assignment does not change the matrix values, therefore the part of `Pre` concerning the matrix stays true. The problem statement guarantees that the character read is either **‘S’** or **‘M’**, thus the first conjunct of `Pre` is also preserved.  

*Result*: after line 1 the invariant `Pre` still holds.

---

### 3.2. Building the matrix

```
{Pre}
mat = [[float(input()) for _ in range(12)] for _ in range(12)]
{  ∀i,j : mat[i][j] = a[i][j]  ∧  op ∈ {'S','M'} }
```

The comprehension reads exactly 12·12 numbers, each converted to a `float`. By construction each entry `mat[i][j]` equals the corresponding input value `a[i][j]`. No other variable is modified, therefore the condition on `op` is unchanged.

*Result*: after line 3 we have a concrete matrix `mat` that satisfies the matrix part of `Pre`.

### 3.3. Initialising the accumulator

{soma = 0}
{  soma = 0 }
```

A trivial assignment; the invariant we will need for the loops is introduced now.

### 3.4. Loop invariants

#### 3.4.1. Outer loop (`i`)

We treat the outer `for` as a `while` loop with a counter `i` ranging from `1` to `11`.  
**Invariant I₁** (holds at the start of each outer‑iteration):

```
I₁(i)  ≡  1 ≤ i ≤ 12   ∧
          soma = Σ_{p=1}^{i‑1} Σ_{q=12‑p}^{11} mat[p][q]
```

*Explanation*: after finishing the iteration for a given `i‑1`, `soma` already contains the sum of all elements that are **below the secondary diagonal** in rows `1 … i‑1`. (Row `0` has no element below the secondary diagonal, therefore the sum starts at `p = 1`.)

**Initialization** (`i = 1` before the first iteration):

```
i = 1  →  the outer range is empty ( Σ over p = 1..0 = 0 )
soma = 0
```

Hence `I₁(1)` holds because the double sum is zero.

**Maintenance**: Assume `I₁(i)` holds at the beginning of the iteration for a particular `i` (`1 ≤ i ≤ 11`).  
During this iteration the inner loop adds *exactly* the elements `mat[i][j]` with `j` ranging from `11` down to `12‑i`.  
These are precisely the columns that satisfy `j > 11‑i`, i.e. `BelowSec(i,j)`.  

After the inner loop finishes we have

```
soma' = soma + Σ_{j=12‑i}^{11} mat[i][j]
      = Σ_{p=1}^{i‑1} Σ_{q=12‑p}^{11} mat[p][q]  +  Σ_{j=12‑i}^{11} mat[i][j]
      = Σ_{p=1}^{i}   Σ_{q=12‑p}^{11} mat[p][q]
```

Thus the invariant `I₁(i+1)` holds for the next value of `i`.

**Termination**: The outer `for` stops when `i = 12`. At that point `I₁(12)` gives

```
soma = Σ_{p=1}^{11} Σ_{q=12‑p}^{11} mat[p][q]   (since for p = 12 the inner range is empty)
```

which is exactly the required sum `S` (the definition of `S` excludes row 0 because there is no element below the secondary diagonal in that row).

Hence after line 9 we have established

```
{ soma = S }
```

---

#### 3.4.2. Inner loop (`j`)

For completeness we also give the invariant for the inner loop, although it is *derived* from `I₁`:

**Invariant I₂(i,j)** (holds at the start of each inner‑iteration for a fixed outer `i`):

```
I₂(i,j)  ≡  11‑i < j ≤ 11   ∧
           soma = Σ_{p=1}^{i‑1} Σ_{q=12‑p}^{11} mat[p][q]  +
                  Σ_{k=j+1}^{11} mat[i][k]
```

*Initialization*: when the inner loop starts, `j = 11`. The second sum is empty, so `I₂(i,11)` reduces to `I₁(i)` – which holds by the outer invariant.

*Maintenance*: the body `soma += mat[i][j]` adds `mat[i][j]` to `soma` and then decrements `j`. Algebraically the invariant is preserved exactly as shown in the outer‑loop maintenance proof.

*Termination*: when the inner loop exits we have `j = 11‑i`. Substituting this value into `I₂` makes the second sum become the full range `Σ_{k=12‑i}^{11} mat[i][k]`, which yields the updated outer invariant `I₁(i+1)`.

Thus both invariants are sound.

### 3.5. The conditional output

After the loops we know `soma = S`. The final `if`‑statement is:

```
if op == 'S':
    print(f'{soma:.1f}')
else:
    print(f'{soma/66:.1f}')
```

We prove the two branches separately.

*Branch 1* (`op = 'S'`):

```
{ op = 'S' ∧ soma = S }
print(f'{soma:.1f}')
{ output = S }
```

`print` does not change program state except for the observable output; the format `:.1f` merely rounds to one decimal place, which is allowed by the specification (the post‑condition does not prescribe a particular rounding method, only the numeric value). Hence the post‑condition `op='S' → output = S` holds.

*Branch 2* (`op ≠ 'S'`, i.e. `op = 'M'`):

```
{ op = 'M' ∧ soma = S }
print(f'{soma/66:.1f}')
{ output = S/66 }
```

Again the only effect is the observable output, and we have exactly the required average (`66` is the number of elements below the secondary diagonal). Thus the post‑condition `op≠'S' → output = S/66` holds.

By the **conditional rule** of Hoare logic we obtain

```
{ op ∈ {'S','M'} ∧ soma = S }  if … else …  { Post(op, S) }
```

### 3.6. Composition of the whole program

Putting together the three major fragments (reading `op`, building the matrix, the loops, and the conditional) using the *sequence rule* we obtain the final Hoare triple:

```
{ Pre }
   op = input();
   mat = [[float(input()) for _ in range(12)] for _ in range(12)];
   soma = 0;
   for i in range(1,12):
       for j in range(11, 11-i, -1):
           soma += mat[i][j];
   if op == 'S':
       print(f'{soma:.1f}')
   else:
       print(f'{soma/66:.1f}')
{ Post(op, S) }
```

Since each component has been proved correct with respect to its own pre‑ and post‑conditions, the composed program satisfies the specification.
