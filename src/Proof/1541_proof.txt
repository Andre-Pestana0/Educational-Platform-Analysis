## 2.  Loop invariant

The only loop in the program is the outer `while True`.  
Let us denote by `I` the following invariant, which holds **at the beginning of
each iteration** (i.e. just before the test `if caso_teste == "0":` is executed).

```
I  ≡  (the program has processed all previously read lines that were ≠ "0"
       and printed for each of them the correct integer side length)
```

More formally, if the input consists of the sequence of lines  

```
L1 , L2 , … , Lm , "0" , L_{m+2}, …
```

then after the execution of the body of the loop for the first `i` iterations
(`1 ≤ i ≤ m`) we have

```
∀ j (1 ≤ j ≤ i) .  Lj ≠ "0"  ⇒  printed_j = ⌊ √( (A_j·B_j)·100 / C_j ) ⌋ .
```

The invariant also contains the obvious “no‑side‑effects” part:

```
the variables used in the next iteration (caso_teste, A, B, C, …) are
fresh – they will be overwritten by the next input line.
```

## 3.  Proof of the invariant

We prove the invariant by **induction over the number of loop iterations**.

### 3.1  Base case (first iteration)

Before the first execution the invariant is vacuously true because *no* line
has yet been processed.  Hence `I` holds.

### 3.2  Inductive step

Assume that `I` holds at the start of some iteration `k` (induction hypothesis).
We must show that after executing the whole body of the loop the invariant
holds again for iteration `k+1`.

The body consists of the following statements (written in a Hoare‑style
pseudocode, the actual Python syntax being irrelevant for the proof).

```
1  caso_teste := input()
2  if caso_teste == "0" then
3       break
4  else
5       A, B, C := map(int, caso_teste.split())
6       p        := C / 100
7       area_casa:= A * B
8       area_terreno := area_casa / p
9       lado_terreno := sqrt(area_terreno)
10      out := int(lado_terreno)   // printed
```

We verify each part using the Hoare assignment rule:

* **(1) input** – reads a line; the only guarantee needed is that the line is a
  string, which is always true.
* **(2‑3) termination test** – if the line is `"0"` the loop is exited; the
  post‑condition `Q0` requires exactly this behaviour, so the rule is satisfied.
* **(5) parsing** – because `caso_teste ≠ "0"` we know it contains three
  integer tokens separated by blanks.  After the assignment we have the
  relations  

  ```
  A = int(tok1) ,  B = int(tok2) ,  C = int(tok3)
  ```
  and `C > 0` (the problem guarantees a positive percentage).

* **(6) – (9)** – straightforward arithmetic; we can write the derived equalities
  that will be used later:

  ```
  p            = C/100                     (real)
  area_casa    = A·B
  area_terreno = (A·B) / (C/100) = (A·B)·100 / C
  lado_terreno = √area_terreno
  ```

* **(10) printing** – `out = int(lado_terreno) = ⌊ lado_terreno ⌋`.

Putting the equalities together we obtain

```
out = ⌊ √( (A·B)·100 / C ) ⌋ .
```

Thus the value printed in this iteration satisfies exactly the specification
required for the current line `caso_teste`.  Consequently the invariant `I`
holds after the body: all lines processed so far (including the current one)
have been printed correctly, and the variables for the next iteration are ready
to be overwritten by the next input line.  

Hence `I` is preserved.

## 4.  Termination

The loop is `while True`.  Termination follows from the **input‑termination
assumption**: the input stream is finite and contains a line `"0"` (the problem
statement explicitly says “the execution terminates when the line `0` is read”).
Because each iteration reads exactly one line, after a finite number of
iterations the condition `caso_teste == "0"` becomes true and the `break`
statement is executed.  Thus the loop terminates.

Formally, we can view the loop as a **while‑loop with a variant** `n`,
the number of lines remaining (including the current one) in the input.
`n` is a natural number that strictly decreases at each iteration, therefore
the loop cannot execute indefinitely.

## 5.  Whole program correctness

We now combine the pieces using the Hoare composition rule.

1. **Outer loop**  

   ```
   {P0} while True do { I }
   {I ∧ (caso_teste == "0")} break
   {I ∧ (caso_teste == "0")}   // loop exits
   ```

   By the invariant proof we have `{I} body {I}`; by the variant argument the
   loop terminates, so the Hoare rule for `while` gives `{P0} while True … {I}`.

2. **Exit condition**  

   When the loop exits we know `caso_teste == "0"`.  No further output is
   produced, which matches the part of `Q0` that says “the program stops after
   reading the first line equal to `0`”.

3. **Post‑condition**  

   The invariant `I` guarantees that for every processed line (i.e. for every
   line different from `"0"`) the printed value equals  

   ```
   ⌊ √( (A·B)·100 / C ) ⌋ .
   ```

   This is precisely the required output.  Consequently `{P0} program {Q0}` holds.

Thus the Hoare triple

```
{ true }   (the whole program)   { Q0 }
```

is proved, i.e. **the program is correct with respect to the given
pre‑ and post‑conditions**.
