Below is a **complete Hoare‑logic proof** that the program you posted satisfies the natural specification of the problem.  
The proof is written in a “textbook style’’: we first state the **pre‑ and post‑conditions**, then we give a **loop invariant** for the outer `for`‑loop, we prove that the invariant holds initially, that it is preserved by one iteration, and finally that it yields the required post‑condition when the loop terminates.  
All auxiliary facts (division algorithm, integer arithmetic, etc.) are proved as tiny lemmas that are used inside the main proof.

---

## 1.  Language, notation and elementary lemmas  

*We work in a simple imperative language that contains the statements used in the program*  

```
x = e               (assignment)
S1 ; S2             (sequencing)
while B do S end   (while‑loop)
```

*The semantics of the language is the usual one (total‑correctness Hoare logic).*

### Lemma 1 (Division algorithm)  

For every pair of integers `N ≥ 0` and `K > 0` there exist unique integers  

```
q = N // K      (integer division, i.e. floor(N/K))
r = N % K       (remainder)
```

such that  

```
N = K·q + r   and   0 ≤ r < K .
```

*Proof.* Standard number‑theoretic fact, proved by Euclid’s algorithm. ∎



### Lemma 2 (Arithmetic rewriting)  

If `q = N // K` and `r = N % K` then  

```
(N - N % K) / K = q .
```

*Proof.* By Lemma 1, `N - N % K = K·q`. Dividing by `K` (exact division, because the numerator is a multiple of `K`) yields `q`. ∎

## 2.  Specification  

### 2.1  Variables  

* Input variables (read from `stdin`):
  * `T` – number of test cases (`T ≥ 0`);
  * For each iteration `i` (0‑based) we read a pair `N_i , K_i` with `N_i ≥ 0` and `K_i > 0`.

* Output: the program prints, for every test case `i`, a single integer  

```
out_i = N_i % K_i + (N_i - N_i % K_i) / K_i .
```

Because of Lemma 2 this value is equal to  

```
out_i = (N_i // K_i) + (N_i % K_i) .
```

### 2.2  Pre‑condition  

```
P  ≡  (T ∈ ℕ) ∧ (∀ i . 0 ≤ i < T ⇒ (N_i ∈ ℕ ∧ K_i ∈ ℕ_{>0}))
```

In words: the first line of the input is a non‑negative integer `T`, and every following line contains two positive integers `N_i , K_i`.

### 2.3  Post‑condition  

```
Q  ≡  (∀ i . 0 ≤ i < T ⇒ printed(i) = N_i % K_i + (N_i - N_i % K_i)/K_i)
```

`printed(i)` denotes the integer that the program writes on the `i`‑th line of its output.  

Thus the program must output exactly the expression required by the problem statement for each test case.

## 3.  Proof outline  

The program consists of three parts:

1. `T = int(input())`                                            (assignment)
2. `i = 0` (initialisation of the loop counter)                  (assignment)
3. `while i < T do … end`                                         (while‑loop)

The body of the `while`‑loop is itself a sequence of statements:

```
N, K = [int(j) for j in input().split()]      (simultaneous assignment)
print( int( N % K + (N - N % K)/K ) )         (output)
i = i + 1                                      (counter increment)
```

We shall prove **total correctness** (termination + partial correctness) using the following invariant.

### Loop invariant  

```
I(i)  ≡  (0 ≤ i ≤ T) ∧
        (∀ j . 0 ≤ j < i ⇒ printed(j) = N_j % K_j + (N_j - N_j % K_j)/K_j)
```

*Interpretation*: after the loop has executed `i` times, exactly the first `i` test cases have already been processed and the output printed for each of them satisfies the required formula.

## 4.  Detailed Hoare derivation  

### 4.1  The first assignment  

```
{ P }   T = int(input())   { T ≥ 0 }
```

The statement reads an integer from the input; by the definition of `P` this integer is non‑negative, therefore the post‑condition of the assignment is exactly `T ≥ 0`.  
(We keep the rest of `P` (the existence of the `N_i , K_i` pairs) untouched – it is a *global* condition that does not involve `T`.)

### 4.2  Initialisation of the loop counter  

```
{ T ≥ 0 }   i = 0   { I(0) }
```

*Proof*: `i = 0` makes the first conjunct `0 ≤ i ≤ T` true because `T ≥ 0`.  
The second conjunct quantifies over `j` with `0 ≤ j < 0`, i.e. over an empty set; it is therefore vacuously true. ∎

### 4.3  Preservation of the invariant  

We must show  

```
{ I(i) ∧ i < T }   body   { I(i+1) }
```

The body is a *sequence* `S1 ; S2 ; S3` where  

* `S1` – read the next pair `N, K`;
* `S2` – print the expression;
* `S3` – `i = i + 1`.

We treat each component.

#### 4.3.1  Reading the pair  

```
{ I(i) ∧ i < T }   N, K = …   { I(i) ∧ i < T ∧ N = N_i ∧ K = K_i }
```

The input line that is read is exactly the `(i)`‑th line of the test‑case section, therefore the values assigned to `N` and `K` are precisely `N_i` and `K_i`. No variable that appears in `I(i)` is modified, so the invariant part of the assertion stays unchanged.

#### 4.3.2  The `print` statement  

We need the Hoare triple

```
{ I(i) ∧ i < T ∧ N = N_i ∧ K = K_i }   print( int(N % K + (N - N % K)/K) )
{ I(i) ∧ i < T ∧ printed(i) = N_i % K_i + (N_i - N_i % K_i)/K_i }
```

*Proof*:  
The expression that is printed is exactly `N % K + (N - N % K)/K`.  
Using Lemma 2 with `N = N_i` and `K = K_i` we have  

```
(N_i - N_i % K_i)/K_i = N_i // K_i .
```

Hence the printed value equals  

```
N_i % K_i + N_i // K_i = N_i % K_i + (N_i - N_i % K_i)/K_i .
```

The `print` statement does not modify any program variable; it only adds a line to the output. Consequently the first conjunct of `I(i)` (the range of `i`) remains true, and the second conjunct is extended with the freshly printed line for index `i`. ∎

#### 4.3.3  Counter increment  

```
{ I(i) ∧ i < T ∧ printed(i) = … }   i = i + 1   { I(i+1) }
```

The assignment updates `i` to `i+1`.  
* The new value obviously satisfies `0 ≤ i+1 ≤ T` because the guard `i < T` held before the assignment.
* The second conjunct of `I(i+1)` quantifies over `j` with `0 ≤ j < i+1`.  
  – For `j < i` we already have the required equality from the induction hypothesis `I(i)`.  
  – For `j = i` we have just proved it in the previous step (`print`).  

Therefore `I(i+1)` holds after the increment. ∎

Putting the three sub‑proofs together (sequencing rule) we obtain the preservation triple for the whole body.

### 4.4  Termination of the loop  

The loop guard is `i < T`.  
The variant function is `T - i`, which is a non‑negative integer that **strictly decreases** after each iteration (because of `i = i + 1`).  
Since it cannot decrease indefinitely, the `while` loop is guaranteed to terminate.  

Thus we have a *total‑correctness* proof for the loop:

```
{ I(0) }  while i < T do body end  { I(T) }
```

### 4.5  Post‑condition after the loop  

From `I(T)` we obtain:

* `0 ≤ T ≤ T` – trivial;
* For every `j` with `0 ≤ j < T` we have  

```
printed(j) = N_j % K_j + (N_j - N_j % K_j)/K_j .
```

This is exactly the post‑condition `Q` we wanted.

## 5.  Summary of the proof  

| Program fragment | Pre‑condition | Post‑condition |
|------------------|---------------|----------------|
| `T = int(input())` | `P` | `T ≥ 0 ∧ (∀ i<T . N_i , K_i satisfy the input spec)` |
| `i = 0` | `T ≥ 0` | `I(0)` |
| `while i < T do … end` | `I(0)` | `I(T)` (by invariant preservation + termination) |
| Body of the loop (`N,K` read, `print`, `i=i+1`) | `I(i) ∧ i < T` | `I(i+1)` (shown in §§4.3.1‑4.3.3) |
| End of program | `I(T)` | `Q` (definition of `I(T)`) |

Consequently we have proved the **total correctness** of the whole program with respect to the specification `P → Q`. In plain words:

> *For every non‑negative integer `T` and for every pair of positive integers `(N_i , K_i)` supplied on the following `T` lines, the program prints on line `i` the value `N_i % K_i + (N_i - N_i % K_i)/K_i`, i.e. the sum of the remainder of the division of `N_i` by `K_i` and the integer quotient of that division.*

This matches precisely the intended behaviour of the Beecrowd “Iniciante – Nível 1’’ problem, and the Hoare‑logic proof demonstrates that the program is **correct**. ∎
