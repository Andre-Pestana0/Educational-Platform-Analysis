**Answer – a Hoare‑logic style proof that the program meets its specification**

Below we give a complete correctness argument for the program that you posted.  
The argument follows the usual Hoare‑logic proof pattern:

1. **State the specification (pre‑ and post‑conditions).**  
2. **Introduce the loop invariants** for the two loops (the outer `while True` and the inner `for`).  
3. **Show that the invariants are established, preserved and, together with the exit‑conditions, imply the post‑condition.**  
4. **Conclude** that the whole program satisfies the Hoare triple  

\[
\{P\}\;{\tt program}\;\{Q\}.
\]

---

## 1. Specification

We write `input` for the (potentially infinite) sequence of lines that the program reads
from standard input.  Each line is a non‑empty string that we denote by `ℓ`.  
The line is split on the character ‘‑’; the resulting list  

```
texto = ℓ.split('-')
```

is a finite sequence of non‑empty words  

\[
texto = \langle w_1,\dots ,w_n\rangle ,
\qquad  n\ge 1 .
\]

For a word \(w\) we define  

* `first(w) = w[0]` – the first character,
* `last (w) = w[|w|-1]` – the last character,
* `low(c) =` the lower‑case version of the character `c`.

We also fix the target acronym  

\[
\mathsf{TARGET}=``cobol'' .
\]

---

### Pre‑condition \(P\)

The program is invoked with an input stream that may contain any number (including zero) of
lines.  No further assumptions are required:

\[
P \; \equiv \; \text{“the input stream is well‑formed (each line is a string).”}
\]

---

### Post‑condition \(Q\)

For **every** line \(\ell\) that is read, let  

\[
\mathsf{match}(\ell) \; \stackrel{\rm def}{=}\;
\exists\; i_1<\dots<i_6\; \bigl(
   low(first(w_{i_k})) = \mathsf{TARGET}[k]\;
   \lor\;
   low(last (w_{i_k})) = \mathsf{TARGET}[k]\;
   \bigr)_{k=1}^{6}
\]

(i.e. there is a subsequence of six words whose first **or** last letters,
case‑insensitively, spell the word “cobol” in order).

The program must print

* `GRACE HOPPER` **iff** \(\mathsf{match}(\ell)\) holds, and
* `BUG` otherwise,

and then continue with the next line; when the input ends (EOF) it terminates.

Formally

\[
Q \; \equiv \;
\forall\;\ell\in\text{input}:
\begin{cases}
\text{output } ``GRACE\;HOPPER'' &\text{if } \mathsf{match}(\ell)\\[2mm]
\text{output } ``BUG''          &\text{otherwise}
\end{cases}
\;\;\text{and the program finally halts.}
\]

## 2. Loop invariants

The program contains two loops:

* **Outer loop** – reads lines until an `EOFError` is raised.
* **Inner loop** – iterates over the words of a single line.

We give a separate invariant for each.

### 2.1 Outer loop invariant \(I_{out}\)

After processing a (possibly empty) prefix of the input stream we have

\[
I_{out} \; \equiv\;
\text{All lines already read satisfy the post‑condition }Q.
\]

In Hoare‑logic notation:

\[
\{P\}\;\textbf{while True do}\;S\;\textbf{except EOFError break}\;\{I_{out}\}
\]

where \(S\) is the body that reads one line and executes the inner loop.

*Initialization*: before the first iteration no line has been processed, so \(I_{out}\) holds trivially.

*Preservation*: the body reads a line \(\ell\), runs the inner loop, prints the correct
message (proved below) and then proceeds to the next iteration.  Hence after the body
finishes, the newly processed line also satisfies \(Q\); therefore \(I_{out}\) is still true.

*Termination*: the loop terminates only when an `EOFError` is caught.  At that moment
the whole input has been processed, so \(I_{out}\) together with the termination
condition yields exactly the post‑condition \(Q\) for the whole program.

Thus proving the inner loop is enough to establish the whole program’s correctness.

### 2.2 Inner loop invariant \(I_{in}\)

The inner loop iterates over the list `texto = ⟨w₁,…,w_n⟩`.  
Let `cobol` be the variable that accumulates matched letters.  
Define the **prefix function**

\[
\mathsf{pref}(k) = \begin{cases}
\text{the first }k\text{ letters of ``cobol''} & 0\le k\le 6\\
\text{``cobol''}                               & k=6
\end{cases}
\]

The invariant is:

\[
I_{in}(j) \; \equiv\; 
\begin{aligned}
& 0\le |cobol| \le 6,\\
&\exists\,k\; (k = |cobol|) \;\wedge\;
\bigl(\forall\;r<j\;.\;
\bigl(low(first(w_r)) = \mathsf{pref}(k)[k]\; \lor\;
      low(last (w_r)) = \mathsf{pref}(k)[k]\bigr)\bigr) .
\end{aligned}
\]

Intuitively: after processing the first `j‑1` words, `cobol` contains exactly the longest
prefix of the target word that can be obtained from those words, and its length is
`|cobol| = k`.

*Initialization* (`j = 1`): before the first iteration `cobol` is set to the empty
string (`''`).  Hence `|cobol| = 0` and the invariant holds vacuously.

*Preservation*: assume `I_{in}(j)` holds at the start of the iteration that examines
word `w_j`.  Let `k = |cobol|`.  The program computes  

```
i = low(first(w_j))
f = low(last(w_j))
```

and then checks the cascade of `if/elif` statements:

```
if k==0 and (i=='c' or f=='c'): cobol+='c'
elif k==1 and (i=='o' or f=='o'): cobol+='o'
...
elif k==4 and (i=='l' or f=='l'): cobol+='l'
```

Exactly one of the five guards can be true, namely the one that asks whether the
*next* needed letter of the target (`\mathsf{pref}(k+1)[k]`) appears as either the first
or the last character of `w_j`.  There are two cases:

* **Case 1 – guard true**: the required letter is present, the program appends it,
  so `cobol` becomes `\mathsf{pref}(k+1)`.  Consequently `|cobol| = k+1` and the
  invariant holds for `j+1`.

* **Case 2 – no guard true**: the needed letter does **not** appear in `w_j`.  The
  program leaves `cobol` unchanged, i.e. still `\mathsf{pref}(k)`.  Because no
  additional prefix can be built from `w_j`, the invariant also holds for `j+1`.

Thus `I_{in}(j) ⇒ I_{in}(j+1)`.

*Termination*: the loop ends after the last word `w_n` has been examined (`j = n+1`).  
At that point `I_{in}(n+1)` tells us that `cobol` equals the longest prefix of
``cobol`` that can be obtained from **all** words of the line.  Consequently

\[
cobol = ``cobol'' \;\Longleftrightarrow\; \mathsf{match}(\ell).
\]

The subsequent `if cobol == 'cobol': … else: …` therefore prints exactly the
required message.  Hence the inner loop together with the final `if` establishes the
post‑condition for the current line.

## 3. Hoare‑logic derivation

Below we sketch the formal derivation using the standard Hoare rules.

### 3.1 Inner loop

*Rule (while)*  

\[
\frac{\{I_{in}(j)\; \land\; B\}\; S\;\{I_{in}(j+1)\}}
     {\{I_{in}(1)\}\; \textbf{for }j\text{ in }1..n\; S\; \{I_{in}(n+1)\}}
\]

where `B` is the loop guard (`j ≤ n`).  
We have already shown:

* **Init**: `{P}` → `{I_{in}(1)}` because `cobol` is set to `''`.
* **Preservation**: `{I_{in}(j) ∧ (j ≤ n)}` → `{I_{in}(j+1)}` (the case analysis above).
* **Exit condition**: `¬B` is `j = n+1`.  From `I_{in}(n+1)` we obtain the desired
  relationship between `cobol` and `\mathsf{match}(\ell)`.

Thus the inner loop is proved correct.

### 3.2 Outer loop

*Rule (while)*  

\[
\frac{\{I_{out} \land\; \text{no EOF}\}\; S\;\{I_{out}\}}
     {\{P\}\; \textbf{while True do } S\; \textbf{except EOFError break}\; \{I_{out}\}}
\]

* `S` is the whole body that reads a line, runs the inner loop and prints the
  answer.  By the inner‑loop proof we have  

  \[
  \{I_{out}\land\neg\text{EOF}\}\;S\;\{I_{out}\}.
  \]

  (`I_{out}` is simply “all lines processed so far satisfy `Q`”.)  

* When an `EOFError` is raised the loop exits; the invariant together with the fact
  that *all* lines have been processed yields the final post‑condition `Q`.

Thus the outer loop is correct.

### 3.3 Whole program

Applying the **sequence** rule to the composition

```
while True:
    try:
        … inner loop …
    except EOFError:
        break
```

gives the Hoare triple

\[
\{P\}\;{\tt program}\;\{Q\},
\]

which is exactly the specification we wanted to prove.

## 4. Conclusion

* The **inner `for` loop invariant** guarantees that the variable `cobol` always
  contains the longest prefix of the target word that can be built from the words
  examined so far.  The invariant is established before the first iteration,
  preserved by each iteration, and together with the loop‑exit condition yields the
  exact condition `cobol == "cobol"` ⇔ the line contains a subsequence of words that
  spells the acronym *cobol*.

* The **outer `while True` loop invariant** states that every line already processed
  satisfies the required output property.  The body of the loop (the inner loop plus
  the final `if`) maintains this invariant, and the loop terminates precisely when
  the input stream ends (EOF).

* By the Hoare‑logic rules for **initialisation**, **maintenance**, **termination**
  of loops and for **exception‑driven exit**, we have derived the Hoare triple  

  \[
  \{P\}\;{\tt program}\;\{Q\},
  \]

  i.e. the program is *partially correct* with respect to the given specification,
  and because the only way to leave the outer loop is the explicit `EOFError`
  handler, the program is also *totally correct* (it always terminates).

Therefore the program you provided is **correct**: for every input line it prints
`GRACE HOPPER` exactly when the line contains a subsequence of words whose first
or last letters (case‑insensitively) form the word “cobol”, and prints `BUG`
otherwise, and it stops cleanly at end‑of‑file. ∎
