**Proof that the program satisfies the Hoare‚Äëlogic specification**  

We prove the Hoare triple  

\[
\{P\}\; \texttt{solve\_change()}\; \{Q\}
\]

where  

* `solve_change()` is the whole program (read, compute the twelve quantities, print).  
* `P` is the pre‚Äëcondition given in the statement (the input is a non‚Äënegative real `N` with at most two decimal digits, i.e. `c = 100¬∑N ‚àà ‚Ñï`).  
* `Q` is the post‚Äëcondition that describes the twelve quantities `q‚ÇÅ ‚Ä¶ q‚ÇÅ‚ÇÇ` (the number of notes/coins), the exact decomposition of the amount, the optimality of the decomposition, the greedy construction and the required output format.

The proof proceeds by **sequential composition** of Hoare triples for each program fragment.  
Because the program contains no loops, the only non‚Äëtrivial part of the proof is the establishment of the *greedy invariant* after each ‚Äúdivision‚Äëremainder‚Äô‚Äô pair.

## 2.  Proof outline  

1. **Conversion of the input** ‚Äì from a decimal string to the integer `c`.  
2. **Greedy construction** ‚Äì for each denomination `d_i` we compute `q_i = r_{i‚Äë1}//d_i` and `r_i = r_{i‚Äë1} % d_i`.  
   We maintain the invariant  

   \[
   \boxed{I_i \;:\; r_{i-1}= \operatorname{rem}(c,d_1)+d_1q_1+\dots+d_{i-1}q_{i-1}}
   \tag{1}
   \]

   (i.e. after handling the first `i‚Äë1` denominations the current value of `valor`
   is exactly the remainder `r_{i-1}` of the Euclidean division of `c` by `d_i`).  
3. **Termination of the sequence** ‚Äì after the last pair we have `r_{12}=0` and therefore  

   \[
   c = \sum_{i=1}^{12} d_i q_i .
   \tag{2}
   \]

4. **Optimality** ‚Äì we prove that the greedy construction (the values that satisfy (1)) yields a decomposition with the minimum possible total number of pieces.  
5. **Output** ‚Äì we show that the printed lines correspond exactly to the values `q_i` and to the strings required in clause (iv) of `Q`.

## 3.  Detailed Hoare‚Äëlogic derivation  

### 3.1  Input conversion  

```
valor = float(input())
valor = int(valor*100)
```

*Pre‚Äëcondition*  

\[
P \equiv 0\le N\le 10^{6}\land N=\frac{c}{100}\land c\in\mathbb N .
\]

*First assignment*  

\[
\{P\}\; \texttt{valor = float(input())}\; \{P\land \texttt{valor}=N\}
\]

(the `float` conversion does not change the value because the input is exactly `N`).

*Second assignment*  

\[
\{P\land \texttt{valor}=N\}\;
\texttt{valor = int(valor*100)}\;
\{P\land \texttt{valor}=c\}
\]

Indeed `valor*100 = N¬∑100 = c` and `c` is an integer, therefore `int(c)=c`.  
Thus after the two statements we have the **loop‚Äëfree invariant**

\[
I_0 \;:\; \texttt{valor}=c .
\tag{3}
\]

---------------------------------------------------------------------

### 3.2  Greedy step for a generic denomination  

For a fixed `i (1‚â§i‚â§12)` we consider the pair

```
q_i = valor // d_i
valor = valor % d_i
```

Assume that before the pair the invariant `I_{i-1}` holds, i.e.

\[
\texttt{valor}= r_{i-1}
\quad\text{with}\quad
c = d_1 q_1 +\dots + d_{i-1} q_{i-1} + r_{i-1},
\qquad 0\le r_{i-1}< d_i .
\tag{4}
\]

*Division*  

\[
\{I_{i-1}\}\;
\texttt{q_i = valor // d_i}\;
\{I_{i-1}\land \texttt{q_i}= \big\lfloor\frac{r_{i-1}}{d_i}\big\rfloor\}
\tag{5}
\]

*Remainder*  

\[
\{I_{i-1}\land \texttt{q_i}= \big\lfloor\frac{r_{i-1}}{d_i}\big\rfloor\}\;
\texttt{valor = valor % d_i}\;
\{I_i\}
\tag{6}
\]

where  

\[
I_i \;:\; \texttt{valor}= r_i
\quad\text{and}\quad
c = d_1 q_1+\dots + d_i q_i + r_i ,
\qquad 0\le r_i< d_{i+1}.
\]

The equality follows from the definition of Euclidean division:

\[
r_{i-1}= d_i\cdot\big\lfloor\frac{r_{i-1}}{d_i}\big\rfloor + \operatorname{rem}(r_{i-1},d_i)
      = d_i q_i + r_i .
\]

Substituting the expression of `r_{i-1}` given by (4) yields exactly `I_i`.  
Thus the Hoare rule for *composition* (`{A}S‚ÇÅ{B}` and `{B}S‚ÇÇ{C}` ‚áí `{A} S‚ÇÅ;S‚ÇÇ {C}`) gives a valid triple for the pair.

### 3.3  The whole sequence  

Starting from `I‚ÇÄ` (3) and applying the above reasoning for `i = 1,‚Ä¶,12` we obtain the chain  

\[
I_0 \;\xRightarrow{\text{pair }1}\; I_1 \;\xRightarrow{\text{pair }2}\; \dots \;\xRightarrow{\text{pair }12}\; I_{12}.

`I‚ÇÅ‚ÇÇ` is

\[

&\texttt{valor}= r_{12}=0,\\[1mm]

\]

Hence after the twelve ‚Äúdivision‚Äëremainder‚Äô‚Äô statements the program has computed

### 3.4  Minimality of the greedy solution  

The set of denominations  

\[
(10000, 5000, 2000, 1000, 500, 200, 100, 50, 25, 10, 5, 1)
\]

is **canonical**: each denomination is a multiple of the next one,
i.e. `d_i = k_i¬∑d_{i+1}` with `k_i ‚àà ‚Ñï` (for the present set `k_i ‚àà {2,5,2,2,5,2,2,5,2,2,5}`).
For such a system the greedy algorithm is optimal.  
A short exchange argument proves it:

*Lemma.*  
Let `c` be any amount of cents and let `q‚ÇÅ,‚Ä¶,q‚ÇÅ‚ÇÇ` be the greedy quantities.
Suppose another feasible decomposition `q'_1,‚Ä¶,q'_12` exists with the same total value.
If there is an index `j` with `q'_j > q_j`, then because `d_j` is a multiple of `d_{j+1}`
we can replace one coin of denomination `d_j` by at most `k_j` coins of the next
smaller denomination, thereby **increasing** the total number of pieces.
Consequently any decomposition that uses fewer pieces than the greedy one must
have `q'_j ‚â§ q_j` for every `j`, which forces `‚àë q'_i = ‚àë q_i`.

Therefore the greedy construction satisfies clause (ii) of `Q` (minimal total number of pieces).

By the **rule of composition** (if `{A}S‚ÇÅ{B}` and `{B}S‚ÇÇ{C}` then `{A}S‚ÇÅ;S‚ÇÇ{C}`) we obtain

All four clauses of the post‚Äëcondition `Q` have been proved:
