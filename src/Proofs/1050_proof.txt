## 1.  What we have to show  

We are given a **HoareтАСlogic specification**

\[
\{\,\exists d\in\mathbb Z.\;{\tt read}=d\,\}\;\;{\tt prog}\;\;
\{\,out = R(d)\,\},
\qquad
R(d)=\begin{cases}
\Delta(d) & d\in\operatorname{dom}(\Delta)\\[2pt]
\text{тАЬDDD nao cadastradoтАЭ} & d\notin\operatorname{dom}(\Delta)
\end{cases}
\tag{S}
\]

and a concrete Python program (the тАЬsolution of Beecrowd 1050тАЭ).  
Our task is to **prove** that the program satisfies (S).  
We do this by writing a *HoareтАСlogic derivation* that starts from the
preтАСcondition, applies the standard inference rules for the language
constructs that appear in the program, and ends with the postтАСcondition.

The proof consists of two parts  

* **Partial correctness** тАУ the program, if it terminates, produces the
  required output.  
* **Total correctness** тАУ the program always terminates (in particular,
  the `ifтАСelse` terminates).

---

## 2.  The program in a HoareтАСlogic friendly syntax  

For the sake of a formal proof we rewrite the Python fragment as a
sequence of elementary commands of a simple imperative language
(`:=` for assignment, `if b then SтВБ else SтВВ`, `print e`).  The program
contains only **readтАСonce** input, a constant dictionary, a membership
test, a dictionary lookup and a `print`.  All of them are primitive
operations that we treat as *atomic* (they have a known HoareтАСlogic
axiom).

```

2  cidades := { тАж }                # the literal dictionary (see below)
3  if ddd in cidades then
4        out := cidades[ddd]      # the тАЬthenтАЭ branch
5  else

```

The *literal* dictionary in lineтАп2 is exactly the partial function
\(\Delta\) that appears in the specification; we therefore assume the
following **definitionтАСequivalence** (proved once and for all by inspection
of the source code):

\[
\boxed{\;cidades \equiv \Delta\;}
\tag{D}
\]

Consequently

\[
ddd\in cidades \;\;\Longleftrightarrow\;\; ddd\in\operatorname{dom}(\Delta)
\tag{M}
\]

and, for every integer `x` that belongs to the domain,

\[
cidades[x] = \Delta(x). \tag{L}
\]

## 3.  HoareтАСlogic rules that we shall use  

| construct | Hoare rule (partial correctness) |
|-----------|-----------------------------------|
| **Assignment** | \(\displaystyle \frac{}{\{P[e/x]\}\;x:=e\;\{P\}}\) |
| **IfтАСthenтАСelse** | \(\displaystyle \frac{\{P\wedge b\}\;S_1\;\{Q\}\qquad\{P\wedge\neg b\}\;S_2\;\{Q\}}{\{P\}\;\textbf{if }b\textbf{ then }S_1\textbf{ else }S_2\;\{Q\}}\) |
| **Sequence** | \(\displaystyle \frac{\{P\}\;S_1\;\{R\}\qquad\{R\}\;S_2\;\{Q\}}{\{P\}\;S_1;S_2\;\{Q\}}\) |
| **Print** (treated as an assignment to the *output* variable) | \(\displaystyle \frac{}{\{out = v\}\;\textbf{print }v\;\{out = v\}}\) |
| **Read** (atomic input) | \(\displaystyle \frac{}{\{\,read = d\,\}\;\textbf{read}\;d\;\{\,ddd = d\,\}}\) |

The rule for **total correctness** adds a *variant* that strictly
decreases for each loop or recursive call.  Our program contains no
loop, therefore total correctness follows from the fact that each
primitive command terminates (reading an integer, evaluating a literal
dictionary, a membership test and a single `print` are all terminating).

## 4.  Annotated proof  

We annotate each line of the program with a *loopтАСinvariantтАСstyle*
assertion.  The assertions are written as logical formulas over the
variables `ddd`, `cidades`, `out` and the *input* constant `read`.

### 4.1  After the input command (lineтАп1)

```
{ тИГdтИИтДд . read = d }                -- preтАСcondition (2)
read ddd                           -- тАЬreadтАЭ is an atomic command
{ тИГdтИИтДд . read = d тИз ddd = d }     -- by the readтАСaxiom
```

Since the only value that `read` can have is the integer supplied by the
environment, we can rewrite the postтАСcondition of lineтАп1 as

\[
\boxed{P_1 \;:\; ddd = d \;\wedge\; d\in\mathbb Z}
\tag{PтВБ}
\]

where the existential quantifier is now **eliminated** (the program
stores the concrete value `d` in `ddd`).

### 4.2  After the dictionary construction (lineтАп2)

The statement is a *constant* assignment; it does not depend on the
current state, therefore we may strengthen the assertion with the
definition (D):

\[
\frac{\{P_1\}}{cidades := \Delta}\;
\{\,P_1 \;\wedge\; cidades \equiv \Delta\,\}
\stackrel{\text{def}}{=}
\boxed{P_2 \;:\; ddd = d \;\wedge\; cidades = \Delta}
\tag{PтВВ}
\]

(The equality `cidades = ╬Ф` is a shorthand for тАЬthe two partial functions
have exactly the same graphтАЭ.)

### 4.3  The conditional (linesтАп3тАУ6)

We now apply the **ifтАСthenтАСelse** rule with the Boolean guard  

\[
b \;=\; (ddd\in cidades).
\]

Because of (M) we have the logical equivalence  

\[
b \;\Longleftrightarrow\; (d \in \operatorname{dom}(\Delta)).
\tag{B}
\]

We must prove two subтАСderivations:

* **ThenтАСbranch** (`b` true) тАУ lineтАп4  
* **ElseтАСbranch** (`b` false) тАУ lineтАп6  

Both branches end with the same postтАСcondition, which we call `Q`:

\[
Q \;:\; out = 
\begin{cases}
\Delta(d) & d\in\operatorname{dom}(\Delta)\\[2pt]
\text{тАЬDDD nao cadastradoтАЭ} & d\notin\operatorname{dom}(\Delta)
\end{cases}
\tag{Q}
\]

---

### 4.3.1  ThenтАСbranch  

Assume the guard holds, i.e. `ddd тИИ cidades`.  Using (M) we know that
`d тИИ dom(╬Ф)`.  The command in lineтАп4 is the assignment

```
out := cidades[ddd]
```

By (L) the rightтАСhand side equals `╬Ф(d)`.  Applying the assignment rule:

\[
\frac{\{\,Q[╬Ф(d)/out]\}}{out := cidades[ddd]}\;
\{\,Q\,\}
\]

Since `Q[╬Ф(d)/out]` is precisely the assertion  

\[
out = ╬Ф(d) \;\wedge\; d\in\operatorname{dom}(\Delta),
\]

and this follows from `PтВВ` together with the hypothesis `b`,
the thenтАСbranch is proved.

Formally:

\[
\{P_2 \wedge b\}
\;\;out := cidades[ddd]\;
\{Q\}
\tag{T}
\]

---

### 4.3.2  ElseтАСbranch  

Assume the guard is false, i.e. `ddd тИЙ cidades`.  By (M) we obtain
`d тИЙ dom(╬Ф)`.  The command in lineтАп6 is

```

```

Again the assignment rule gives

\[
\frac{\{\,Q["DDD nao cadastrado"/out]\}}{out := "DDD nao cadastrado"}\;
\{\,Q\,\}
\]

and `Q["DDD nao cadastrado"/out]` is exactly

\[
out = \text{тАЬDDD nao cadastradoтАЭ} \;\wedge\; d\notin\operatorname{dom}(\Delta),
\]

which follows from `PтВВ тИз ┬мb`.  Hence

\[
\{P_2 \wedge \neg b\}
\;\;out := "DDD nao cadastrado"\;
\{Q\}
\tag{E}
\]

---

### 4.3.3  Putting the two branches together  

From (T) and (E) the **ifтАСrule** yields

\[
\{P_2\}\;
\textbf{if }(ddd\in cidades)\textbf{ then }out:=cidades[ddd]\textbf{ else }out:="DDD nao cadastrado"\;
\{Q\}.
\tag{IF}
\]

Thus after lineтАп6 we have established the postтАСcondition `Q`.

### 4.4  The final `print` (lineтАп7)

The last statement merely writes the value of `out` to the terminal.
In Hoare logic we treat it as an *output* command that does not change
the logical state, i.e.

\[
\frac{}{\{out = v\}\;\textbf{print }v\;\{out = v\}} .
\]

Instantiating `v` with the current value of `out` we obtain

\[
\{Q\}\;\textbf{print }out\;\{Q\}.
\tag{PR}
\]

---

### 4.5  Whole program  

Now we compose the three pieces:

1. `{тИГd. read = d}` тЖТ (read) тЖТ `PтВБ`.  
2. `PтВБ` тЖТ (dictionary literal) тЖТ `PтВВ`.  
3. `PтВВ` тЖТ (ifтАСelse) тЖТ `Q`.  
4. `Q` тЖТ (print) тЖТ `Q`.

Using the **sequence rule** repeatedly we obtain the Hoare triple

\[
\boxed{\;\{\,\exists d\in\mathbb Z.\;read = d\,\}\;\;{\tt prog}\;\;
\{\,out = R(d)\,\}\;}
\tag{PROV}
\]

where `R(d)` is precisely the definition (1) of the result function.
Thus **partial correctness** is proved.

## 5.  Termination (total correctness)

The program contains no loops or recursion:

* `read` returns a value after a bounded amount of I/O work.  
* The dictionary literal is a compileтАСtime constant; constructing it is a
  finite, bounded operation.  
* The membership test `ddd in cidades` and the lookup `cidades[ddd]`
  are constantтАСtime operations on a finite table.  
* `print` terminates after writing a finite string.

Therefore every execution reaches the final `print` statement in a
finite number of steps.  Consequently the program is **totally
correct** with respect to the specification (5).

---

## 6.  Conclusion  

We have exhibited a complete HoareтАСlogic derivation:

* Starting from the preтАСcondition тАЬthe input is a single integerтАЭ,  
* applying the assignment, ifтАСthenтАСelse and print axioms,  
* using the factual equivalences (D), (M) and (L) that relate the
  concrete Python dictionary to the abstract partial function ╬Ф,  

we have derived exactly the postтАСcondition required by the specification,
namely `out = R(d)`.  The derivation also shows that the program always
terminates, hence it satisfies the **totalтАСcorrectness** Hoare triple

\[
\{\,\exists d\in\mathbb Z.\;read=d\,\}\;{\tt prog}\;
\{\,out = R(d)\,\}.
\]

Consequently the given Python program is **correct** with respect to the
formal HoareтАСlogic specification (5).
